The error "Attempted to check if the value:" is TLC's truncated message for attempting a set operation (`\cap`) on an opaque model value atom. The constants `GWT_IDS` and `VERIFIED_GWTS` were configured as model value atoms, not enumerable sets. The fix is to replace them with integer counts and derive the GWT index sets as integer ranges (`1..N`), which TLC evaluates without any set-membership ambiguity.

    N_GWTS,
    N_VERIFIED,
    MaxSteps

ASSUME N_VERIFIED >= 0 /\ N_VERIFIED <= N_GWTS
ASSUME MaxSteps >= 1

EXTRACT  == "extract"
REGISTER == "register"
LOOP_T   == "loop"
BRIDGE_T == "bridge"
GENTESTS == "gen_tests"

PENDING   == "pending"
COMPLETED == "completed"

GwtIds      == 1..N_GWTS
VerifiedIds == 1..N_VERIFIED

ExtractJob   == {[type |-> EXTRACT]}
RegisterJob  == {[type |-> REGISTER]}
LoopJobs     == {[type |-> LOOP_T,   gwt |-> g] : g \in GwtIds}
