## Issues Found

### Critical

**1. Dead imports — `Edge`, `EdgeType`, `Node`, `NodeKind` are never referenced.**
The `from registry.types import Edge, EdgeType, Node, NodeKind` line imports four names used nowhere in the file. `test_dag_gwt_registration_reflects_behavior` uses only `RegistryDag`. Dead imports silently suggest a test that was never completed.

**2. `TestTrace5.test_idempotent_with_trace3` asserts the wrong condition.**
The test is labelled "idempotent" but only checks `REQUIRED_CORE_SECTIONS <= secs_a` and `REQUIRED_CORE_SECTIONS <= secs_b` independently — two things already tested by the per-trace invariant calls. It never asserts `secs_a == secs_b`, so it cannot falsify any non-determinism. `TestTrace10.test_deterministic_sections` does this correctly. Trace 5 should match Trace 10's idiom.

**3. `TestInvariantTypeOK.test_error_class_in_error_classes` never calls the SUT.**
```python
def test_error_class_in_error_classes(self, error_class, has_cx):
    assert error_class in ERROR_CLASSES
```
Parameters are `("unknown", False)` and `("constant_mismatch", True)` — both are module-level constants defined in `ERROR_CLASSES` three lines above. The assertion is a tautology. The TypeOK invariant intent is that the builder processes valid inputs without escaping its declared type domain; the test should actually call the builder and verify runtime behaviour.

### Moderate

**4. `_has_retry_header` and `closing` detection are proxy-only — the limitation is not documented.**
`_has_retry_header` returns `True` for any non-empty string. `closing` is inferred circularly: if all other required sections are detected, `closing` is assumed present regardless of prompt content. This means `RequiredSectionsPresent` can never falsify a missing `closing` section. These proxies are acceptable (the builder API has no closing-sentinel parameter), but they must be clearly labelled so reviewers do not treat them as true content checks.

**5. `TestTrace5.test_idempotent_with_trace3` name mismatch.** Even after fixing the assertion, the label "idempotent" conflicts with the name used in Trace 10 ("deterministic"). Use `test_deterministic_sections_match_trace3` for clarity.

### Minor

**6. `TestTrace8.test_idempotent_with_trace6` has the same weakness as Trace 5** — it checks section membership individually rather than `secs_a == secs_b`. Fix for consistency.

def _has_initial_prompt(prompt: str) -> bool:
    return _INITIAL_PROMPT in prompt


def _has_previous_output(prompt: str) -> bool:
    return _PREVIOUS_OUTPUT in prompt


def _has_error_message(prompt: str) -> bool:
    return _ERROR_MESSAGE in prompt


def _has_tlc_output(prompt: str) -> bool:
    return _TLC_OUTPUT in prompt


def _has_retry_header(prompt: str) -> bool:
    # PROXY ONLY: the builder does not expose a retry-header sentinel parameter,
    # so we cannot detect it by content.  Per the spec the retry header is
    # always the first step; any non-empty output implies it was emitted.
    # This proxy cannot falsify a missing retry-header but is the best
    # observable signal available without inspecting the builder's internals.
    return prompt is not None and len(prompt.strip()) > 0


def _has_classification(prompt: str, error_class: str) -> bool:
    """Classification block is present iff the error-class instruction string
    appears in the prompt.  Sentinels are chosen to contain no error-class
    names, so this check cannot produce a false positive from sentinel content."""
    return error_class in prompt


def _has_counterexample(prompt: str) -> bool:
    return _COUNTEREXAMPLE in prompt


def _detected_sections(prompt: str, error_class: str, has_counterexample: bool) -> frozenset:
    """
    Build the Python-side equivalent of the TLA+ `prompt_sections` set from
    the actual prompt string returned by build_retry_prompt.

    NOTE – closing detection is a proxy:
      Because the builder exposes no closing-sentinel parameter we cannot
      detect closing by content inspection.  We infer its presence from the
      completion condition: if all other required sections are detected the
      builder reached the Finish state and must have appended the closing
      block.  This means RequiredSectionsPresent cannot falsify a missing
      closing section independently; all other required-section assertions
      provide the observable coverage.
    """
    secs = set()
    if _has_initial_prompt(prompt):
        secs.add("initial_prompt")
    if _has_retry_header(prompt):
        secs.add("retry_header")
    if _has_classification(prompt, error_class) and error_class in INSTRUCTION_CLASSES:
        secs.add("classification")
    if _has_previous_output(prompt):
        secs.add("previous_output")
    if _has_error_message(prompt):
        secs.add("error_message")
    if _has_counterexample(prompt):
        secs.add("counterexample")
    if _has_tlc_output(prompt):
        secs.add("tlc_output")
    # Closing inferred from completion; see docstring above.
    if REQUIRED_CORE_SECTIONS - {"closing"} <= secs:
        secs.add("closing")
    return frozenset(secs)

## Issues Found

### Critical

**1. Dead imports — `Edge`, `EdgeType`, `Node`, `NodeKind` are never referenced.**
The `from registry.types import Edge, EdgeType, Node, NodeKind` line imports four names used nowhere in the file. `test_dag_gwt_registration_reflects_behavior` uses only `RegistryDag`. Dead imports silently suggest a test that was never completed.

**2. `TestTrace5.test_idempotent_with_trace3` asserts the wrong condition.**
The test is labelled "idempotent" but only checks `REQUIRED_CORE_SECTIONS <= secs_a` and `REQUIRED_CORE_SECTIONS <= secs_b` independently — two things already tested by the per-trace invariant calls. It never asserts `secs_a == secs_b`, so it cannot falsify any non-determinism. `TestTrace10.test_deterministic_sections` does this correctly. Trace 5 should match Trace 10's idiom.

**3. `TestInvariantTypeOK.test_error_class_in_error_classes` never calls the SUT.**
```python
def test_error_class_in_error_classes(self, error_class, has_cx):
    assert error_class in ERROR_CLASSES
```
Parameters are `("unknown", False)` and `("constant_mismatch", True)` — both are module-level constants defined in `ERROR_CLASSES` three lines above. The assertion is a tautology. The TypeOK invariant intent is that the builder processes valid inputs without escaping its declared type domain; the test should actually call the builder and verify runtime behaviour.

### Moderate

**4. `_has_retry_header` and `closing` detection are proxy-only — the limitation is not documented.**
`_has_retry_header` returns `True` for any non-empty string. `closing` is inferred circularly: if all other required sections are detected, `closing` is assumed present regardless of prompt content. This means `RequiredSectionsPresent` can never falsify a missing `closing` section. These proxies are acceptable (the builder API has no closing-sentinel parameter), but they must be clearly labelled so reviewers do not treat them as true content checks.

**5. `TestTrace5.test_idempotent_with_trace3` name mismatch.** Even after fixing the assertion, the label "idempotent" conflicts with the name used in Trace 10 ("deterministic"). Use `test_deterministic_sections_match_trace3` for clarity.

### Minor

**6. `TestTrace8.test_idempotent_with_trace6` has the same weakness as Trace 5** — it checks section membership individually rather than `secs_a == secs_b`. Fix for consistency.

    return error_class in prompt

def _detected_sections(prompt: str, error_class: str, has_counterexample: bool) -> frozenset:
    """
    Build the Python-side equivalent of the TLA+ `prompt_sections` set from
    the actual prompt string returned by build_retry_prompt.

    NOTE – closing detection is a proxy:
      Because the builder exposes no closing-sentinel parameter we cannot
      detect closing by content inspection.  We infer its presence from the
      completion condition: if all other required sections are detected the
      builder reached the Finish state and must have appended the closing
      block.  This means RequiredSectionsPresent cannot falsify a missing
      closing section independently; all other required-section assertions
      provide the observable coverage.
    """
    secs = set()
    if _has_initial_prompt(prompt):
        secs.add("initial_prompt")
    if _has_retry_header(prompt):
        secs.add("retry_header")
    if _has_classification(prompt, error_class) and error_class in INSTRUCTION_CLASSES:
        secs.add("classification")
    if _has_previous_output(prompt):
        secs.add("previous_output")
    if _has_error_message(prompt):
        secs.add("error_message")
    if _has_counterexample(prompt):
        secs.add("counterexample")
    if _has_tlc_output(prompt):
        secs.add("tlc_output")
    # Closing inferred from completion; see docstring above.
    if REQUIRED_CORE_SECTIONS - {"closing"} <= secs:
        secs.add("closing")
    return frozenset(secs)
