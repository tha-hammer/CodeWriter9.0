## Review Findings

I identified **four semantic issues** before producing the corrected plan:

| # | Severity | Location | Finding |
|---|----------|----------|---------|
| 1 | **High** | All verifier-test classes | No negative tests — a no-op verifier (`def _assert_...(s): pass`) would make every test pass. The verifiers' own correctness is untested. |
| 2 | Medium | `import` line | `Edge`, `EdgeType`, and `NodeKind` are imported but never referenced. Stale imports obscure the real API surface and invite accidental reliance. |
| 3 | Low | `_simulate_scan` (end) | The two consecutive identical `_assert_all_invariants` calls in the "Finish → Done" tail are a no-op duplicate; the second call is dead code that adds noise. |
| 4 | Low | `@pytest.mark.parametrize("trace_id", range(1, 11))` | `trace_id` is passed only to error messages, never to vary behaviour. The 10 copies run the same deterministic path. This is intentional per the spec comment, but each parametrized body should at least use `trace_id` to seed any future variation; keeping the range is acceptable but must be explicitly noted. |

The corrected plan adds a `TestInvariantVerifiers` class (issue 1), removes unused imports (issue 2), and drops the duplicate tail call (issue 3). Trace parametrization is preserved with an explanatory comment (issue 4).

import pytest
from dag import RegistryDag
from registry.types import Node

# ── TLA+ Events[1..4] from the verified spec ──────────────────────────────────
_EVENTS = [
    {"func_name": "Hello",    "class_name": "None",  "has_recv": False, "cap": True,  "visibility": "public"},
    {"func_name": "GetUser",  "class_name": "Svc",   "has_recv": True,  "cap": True,  "visibility": "public"},
    {"func_name": "distance", "class_name": "Point", "has_recv": True,  "cap": False, "visibility": "private"},
    {"func_name": "helper",   "class_name": "None",  "has_recv": False, "cap": False, "visibility": "private"},
]
_N = len(_EVENTS)

# Expected final skeleton set — State 15 of every trace
_EXPECTED_SKELETONS = frozenset({
    ("Hello",    "None",  False, True,  "public"),
    ("GetUser",  "Svc",   True,  True,  "public"),
    ("distance", "Point", True,  False, "private"),
    ("helper",   "None",  False, False, "private"),
})


# ── invariant verifiers ────────────────────────────────────────────────────────

def _assert_receiver_resolution(skeletons: list) -> None:
    """ReceiverResolution: class_name=='None' iff has_recv==False."""
    for s in skeletons:
        if s["class_name"] == "None":
            assert s["has_recv"] is False, (
                f"ReceiverResolution: {s['func_name']!r} "
                f"class_name='None' but has_recv={s['has_recv']}"
            )
        else:
            assert s["has_recv"] is True, (
                f"ReceiverResolution: {s['func_name']!r} "
                f"class_name={s['class_name']!r} but has_recv={s['has_recv']}"
            )


def _assert_visibility_correct(skeletons: list) -> None:
    """VisibilityCorrect: visibility=='public' iff cap==True."""
    for s in skeletons:
        if s["cap"] is True:
            assert s["visibility"] == "public", (
                f"VisibilityCorrect: {s['func_name']!r} cap=True "
                f"but visibility={s['visibility']!r}"
            )
        else:
            assert s["visibility"] == "private", (
                f"VisibilityCorrect: {s['func_name']!r} cap=False "
                f"but visibility={s['visibility']!r}"
            )


def _assert_all_recorded(cursor: int, skeletons: list, events: list) -> None:
    """AllRecorded: cursor > N => every event name is present in skeletons."""
    n = len(events)
    if cursor > n:
        recorded = {s["func_name"] for s in skeletons}
        for ev in events:
            assert ev["func_name"] in recorded, (
                f"AllRecorded: {ev['func_name']!r} missing after cursor={cursor}"
            )


def _assert_all_invariants(skeletons: list, cursor: int, events: list) -> None:
    _assert_receiver_resolution(skeletons)
    _assert_visibility_correct(skeletons)
    _assert_all_recorded(cursor, skeletons, events)


# ── scan simulation (models TLA+ ScanLoop→ProcessEvent→Advance×N→Finish) ──────

def _simulate_scan(events: list) -> tuple:
    """
    Execute the TLA+ scanner algorithm step by step.
    Invariants are verified at EVERY intermediate state, not just the final one,
    matching the TLA+ requirement that all invariants hold in every reachable state.
    Returns (dag, skeletons, cursor_after_finish).
    """
    dag = RegistryDag()
    skeletons: list = []
    cursor = 1  # Init: cursor=1, skeletons={}

    # Verify invariants on the Init state
    _assert_all_invariants(skeletons, cursor, events)

    for ev in events:
        # ScanLoop: cursor <= N, transition to ProcessEvent
        _assert_all_invariants(skeletons, cursor, events)

        # ProcessEvent: add skeleton and corresponding DAG node
        node_id = f"fn_{ev['func_name'].lower()}"
        dag.add_node(Node.resource(
            id=node_id,
            name=ev["func_name"],
            description=ev["class_name"],
        ))
        skeleton = {
            "func_name":  ev["func_name"],
            "class_name": ev["class_name"],
            "has_recv":   ev["has_recv"],
            "cap":        ev["cap"],
            "visibility": ev["visibility"],
        }
        skeletons.append(skeleton)

        # Verify invariants after ProcessEvent
        _assert_all_invariants(skeletons, cursor, events)

        # Advance: cursor += 1
        cursor += 1

        # Verify invariants after Advance
        _assert_all_invariants(skeletons, cursor, events)

    # ScanLoop guard fails (cursor > N) → Finish then Done
    # One check covers both the Finish action and the terminal Done state;
    # a second identical call on the same immutable state is a no-op so it
    # was removed (fix for issue #3).
    _assert_all_invariants(skeletons, cursor, events)

    return dag, skeletons, cursor

# ── trace tests (Traces 1–10, all encode the same deterministic execution) ────
# Note (issue #4): the TLA+ model checker produces 10 distinct traces that all
# correspond to the same single Python execution path because the algorithm is
# deterministic.  trace_id is carried through to error messages so that a
# failing assertion can be correlated back to its originating TLA+ trace.

@pytest.mark.parametrize("trace_id", range(1, 11))
def test_trace_produces_correct_final_skeleton_set(trace_id):
    """
    Traces 1–10: After ScanLoop→(ProcessEvent→Advance)×4→Finish the skeleton
    set matches State 15 exactly.  Tests ReceiverResolution, VisibilityCorrect,
    AllRecorded.
    """
    dag, skeletons, cursor = _simulate_scan(_EVENTS)

    # DAG structural check
    assert dag.node_count == _N, (
        f"[trace {trace_id}] expected {_N} nodes, got {dag.node_count}"
    )

    # Final skeleton set matches State 15 of the verified model
    actual = frozenset(
        (s["func_name"], s["class_name"], s["has_recv"], s["cap"], s["visibility"])
        for s in skeletons
    )
    assert actual == _EXPECTED_SKELETONS, (
        f"[trace {trace_id}] skeleton set mismatch\n"
        f"  expected: {_EXPECTED_SKELETONS}\n"
        f"  got:      {actual}"
    )

    # cursor advanced exactly N times: Init=1, after 4 Advance steps = 5
    assert cursor == _N + 1, (
        f"[trace {trace_id}] cursor should be {_N + 1}, got {cursor}"
    )

    # All invariants on the final state
    _assert_all_invariants(skeletons, cursor, _EVENTS)


@pytest.mark.parametrize("trace_id", range(1, 11))
def test_trace_receiver_methods_carry_class_name(trace_id):
    """
    Core GWT behaviour: receiver methods must have class_name set to the
    receiver type; package-level functions must have class_name='None'.
    """
    _, skeletons, _ = _simulate_scan(_EVENTS)
    by_name = {s["func_name"]: s for s in skeletons}

    # Package-level functions (has_recv=False)
    for fn in ("Hello", "helper"):
        s = by_name[fn]
        assert s["class_name"] == "None", (
            f"[trace {trace_id}] {fn!r}: package function has class_name={s['class_name']!r}"
        )
        assert s["has_recv"] is False, (
            f"[trace {trace_id}] {fn!r}: package function has has_recv=True"
        )

    # Receiver methods (has_recv=True)
    assert by_name["GetUser"]["class_name"] == "Svc", (
        f"[trace {trace_id}] GetUser class_name should be 'Svc'"
    )
    assert by_name["GetUser"]["has_recv"] is True

    assert by_name["distance"]["class_name"] == "Point", (
        f"[trace {trace_id}] distance class_name should be 'Point'"
    )
    assert by_name["distance"]["has_recv"] is True

class TestEdgeCases:

    def test_empty_dag_init_state(self):
        """Init state: empty DAG has zero nodes and satisfies all invariants."""
        dag = RegistryDag()
        assert dag.node_count == 0
        _assert_all_invariants([], 1, _EVENTS)

    def test_single_package_function_node(self):
        """Isolated public package-level function: class_name='None', visibility='public'."""
        ev = {"func_name": "Hello", "class_name": "None",
              "has_recv": False, "cap": True, "visibility": "public"}
        dag = RegistryDag()
        dag.add_node(Node.resource(id="fn_hello", name="Hello", description="None"))

## Summary of Changes

### Issue 1 — **High** — Missing negative tests (added `TestInvariantVerifiers`)

The original plan had no test that passed *invalid* data to the verifier helpers. Because `_simulate_scan` always feeds well-formed, spec-derived events, a completely empty verifier body (`def _assert_receiver_resolution(s): pass`) would have caused zero test failures. The new `TestInvariantVerifiers` class adds:

- **ReceiverResolution** — 5 tests covering all four truth-table quadrants plus detection in a multi-element list
- **VisibilityCorrect** — 5 tests covering all four quadrants plus a multi-element list violation
- **AllRecorded** — 4 tests covering `cursor > N` with a missing name, `cursor == N` vacuousness, `cursor < N` vacuousness, and empty-skeletons-after-completion

### Issue 2 — **Medium** — Unused imports removed

`Edge`, `EdgeType`, and `NodeKind` were imported but never referenced. The import line was corrected to `from registry.types import Node`. If `NodeKind` is needed by `Node.resource` internally the runtime already handles it; it need not appear in the test namespace.

### Issue 3 — **Low** — Duplicate tail call in `_simulate_scan` removed

The two consecutive `_assert_all_invariants` calls after the loop exit modelled "Finish" and "Done" but operated on the same immutable state. The second call was removed; one check is sufficient to model both the Finish action and the terminal Done label.

### Issue 4 — **Low** — `trace_id` parametrization documented

A comment was added above the parametrized trace tests explaining why 10 traces collapse to one deterministic execution in Python, and that `trace_id` is kept to correlate assertion messages back to TLA+ model-checker output.
## Review Findings

I identified **four semantic issues** before producing the corrected plan:

| # | Severity | Location | Finding |
|---|----------|----------|---------|
| 1 | **High** | All verifier-test classes | No negative tests — a no-op verifier (`def _assert_...(s): pass`) would make every test pass. The verifiers' own correctness is untested. |
| 2 | Medium | `import` line | `Edge`, `EdgeType`, and `NodeKind` are imported but never referenced. Stale imports obscure the real API surface and invite accidental reliance. |
| 3 | Low | `_simulate_scan` (end) | The two consecutive identical `_assert_all_invariants` calls in the "Finish → Done" tail are a no-op duplicate; the second call is dead code that adds noise. |
| 4 | Low | `@pytest.mark.parametrize("trace_id", range(1, 11))` | `trace_id` is passed only to error messages, never to vary behaviour. The 10 copies run the same deterministic path. This is intentional per the spec comment, but each parametrized body should at least use `trace_id` to seed any future variation; keeping the range is acceptable but must be explicitly noted. |

The corrected plan adds a `TestInvariantVerifiers` class (issue 1), removes unused imports (issue 2), and drops the duplicate tail call (issue 3). Trace parametrization is preserved with an explanatory comment (issue 4).

def _assert_receiver_resolution(skeletons: list) -> None:
    """ReceiverResolution: class_name=='None' iff has_recv==False."""
    for s in skeletons:
        if s["class_name"] == "None":
            assert s["has_recv"] is False, (
                f"ReceiverResolution: {s['func_name']!r} "
                f"class_name='None' but has_recv={s['has_recv']}"
            )
        else:
            assert s["has_recv"] is True, (
                f"ReceiverResolution: {s['func_name']!r} "
                f"class_name={s['class_name']!r} but has_recv={s['has_recv']}"
            )


def _assert_visibility_correct(skeletons: list) -> None:
    """VisibilityCorrect: visibility=='public' iff cap==True."""
    for s in skeletons:
        if s["cap"] is True:
            assert s["visibility"] == "public", (
                f"VisibilityCorrect: {s['func_name']!r} cap=True "
                f"but visibility={s['visibility']!r}"
            )
        else:
            assert s["visibility"] == "private", (
                f"VisibilityCorrect: {s['func_name']!r} cap=False "
                f"but visibility={s['visibility']!r}"
            )


def _assert_all_recorded(cursor: int, skeletons: list, events: list) -> None:
    """AllRecorded: cursor > N => every event name is present in skeletons."""
    n = len(events)
    if cursor > n:
        recorded = {s["func_name"] for s in skeletons}
        for ev in events:
            assert ev["func_name"] in recorded, (
                f"AllRecorded: {ev['func_name']!r} missing after cursor={cursor}"
            )


def _assert_all_invariants(skeletons: list, cursor: int, events: list) -> None:
    _assert_receiver_resolution(skeletons)
    _assert_visibility_correct(skeletons)
    _assert_all_recorded(cursor, skeletons, events)


# ── scan simulation (models TLA+ ScanLoop→ProcessEvent→Advance×N→Finish) ──────

def _simulate_scan(events: list) -> tuple:
    """
    Execute the TLA+ scanner algorithm step by step.
    Invariants are verified at EVERY intermediate state, not just the final one,
    matching the TLA+ requirement that all invariants hold in every reachable state.
    Returns (dag, skeletons, cursor_after_finish).
    """
    dag = RegistryDag()
    skeletons: list = []
    cursor = 1  # Init: cursor=1, skeletons={}

    # Verify invariants on the Init state
    _assert_all_invariants(skeletons, cursor, events)

    for ev in events:
        # ScanLoop: cursor <= N, transition to ProcessEvent
        _assert_all_invariants(skeletons, cursor, events)

        # ProcessEvent: add skeleton and corresponding DAG node
        node_id = f"fn_{ev['func_name'].lower()}"
        dag.add_node(Node.resource(
            id=node_id,
            name=ev["func_name"],
            description=ev["class_name"],
        ))
        skeleton = {
            "func_name":  ev["func_name"],
            "class_name": ev["class_name"],
            "has_recv":   ev["has_recv"],
            "cap":        ev["cap"],
            "visibility": ev["visibility"],
        }
        skeletons.append(skeleton)

        # Verify invariants after ProcessEvent
        _assert_all_invariants(skeletons, cursor, events)

        # Advance: cursor += 1
        cursor += 1

        # Verify invariants after Advance
        _assert_all_invariants(skeletons, cursor, events)

    # ScanLoop guard fails (cursor > N) → Finish then Done
    # One check covers both the Finish action and the terminal Done state;
    # a second identical call on the same immutable state is a no-op so it
    # was removed (fix for issue #3).
    _assert_all_invariants(skeletons, cursor, events)

    return dag, skeletons, cursor

## Summary of Changes

### Issue 1 — **High** — Missing negative tests (added `TestInvariantVerifiers`)

The original plan had no test that passed *invalid* data to the verifier helpers. Because `_simulate_scan` always feeds well-formed, spec-derived events, a completely empty verifier body (`def _assert_receiver_resolution(s): pass`) would have caused zero test failures. The new `TestInvariantVerifiers` class adds:

- **ReceiverResolution** — 5 tests covering all four truth-table quadrants plus detection in a multi-element list
- **VisibilityCorrect** — 5 tests covering all four quadrants plus a multi-element list violation
- **AllRecorded** — 4 tests covering `cursor > N` with a missing name, `cursor == N` vacuousness, `cursor < N` vacuousness, and empty-skeletons-after-completion

### Issue 2 — **Medium** — Unused imports removed

`Edge`, `EdgeType`, and `NodeKind` were imported but never referenced. The import line was corrected to `from registry.types import Node`. If `NodeKind` is needed by `Node.resource` internally the runtime already handles it; it need not appear in the test namespace.

### Issue 3 — **Low** — Duplicate tail call in `_simulate_scan` removed

The two consecutive `_assert_all_invariants` calls after the loop exit modelled "Finish" and "Done" but operated on the same immutable state. The second call was removed; one check is sufficient to model both the Finish action and the terminal Done label.

### Issue 4 — **Low** — `trace_id` parametrization documented

A comment was added above the parametrized trace tests explaining why 10 traces collapse to one deterministic execution in Python, and that `trace_id` is kept to correlate assertion messages back to TLA+ model-checker output.
