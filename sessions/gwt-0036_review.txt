## Review Findings

Three semantic defects were identified before producing the corrected plan:

---

### Defect 1 — **Fixture topology inconsistency** (high severity)

The `engine_root` fixture creates the full canonical tree:
```
templates/
  pluscal/
    instances/    ← spec_dir points here
python/
```
But **every standalone test** (the six tests that allocate `tmp_path` directly) only creates `templates/` and `python/`. If `ProjectContext.self_hosting()` validates subdirectory existence, or if any path-derivation logic walks the tree, those tests would either raise at construction time or silently test a different, incomplete environment than the invariant fixture tests. The INSTANCES and SpecInInstancesDir verifiers require the full subtree to be canonical.

**Fix:** Extract a `_make_engine_tree(root)` helper called uniformly everywhere.

---

### Defect 2 — **Speculative API surface in `test_all_engine_derived_paths_descend_from_engine_root`** (medium severity)

The test references `ctx.template_dir`, `ctx.python_dir`, and `ctx.artifact_dir`. None of these attributes appear in the bridge verifiers, the TLA+ variable bindings, or anywhere else in the spec. If these attributes do not exist on `ProjectContext`, the test raises `AttributeError` instead of an informative assertion failure, masking the real problem. The test also adds coverage for attributes never introduced or verified elsewhere.

**Fix:** Guard with `pytest.importorskip`-style `hasattr` checks that produce actionable failure messages, and add an explanatory comment that these are beyond the formal spec boundary.

---

### Defect 3 — **Redundant test** (low severity)

`test_spec_dir_suffix_is_canonical_instances_path` asserts `ctx.spec_dir == ctx.engine_root / "templates" / "pluscal" / "instances"` — identical to `test_spec_dir_equals_instances_path`. Both pass under the same conditions and fail under the same conditions. One is keeping; the exact-match variant is the stronger phrasing, so the suffix test is removed to avoid false confidence from redundant green.
