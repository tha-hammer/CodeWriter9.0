Now I understand the root cause. The `_classify_constants` heuristic defaults constants to **numeric** (value `10`) unless it detects a set-usage pattern. `OldGwts \union OldReqs` doesn't trigger the set classifier — so the cfg assigns `OldGwts = 10`, causing `10 \union 10` to throw a TLC type error at State 1. The fix is `ASSUME X # {}` which the classifier regex explicitly recognises as a set indicator.
Good. All my `define` operators start with `/\`, `phase`, or `step_count` — none start with `{`, `<`, `"`, or a digit — so they all pass through correctly as `INVARIANT` lines. The only fix needed is the constant classification. Here is the corrected, single-copy module:

---- MODULE ExtractPreservesGwts ----
EXTENDS Integers, FiniteSets, TLC

CONSTANTS
    OldGwts,
    OldReqs,
    SchemaNodes,
    MaxSteps

ASSUME OldGwts # {}
ASSUME OldReqs # {}
ASSUME SchemaNodes # {}
