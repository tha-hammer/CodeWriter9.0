def _interface_body_line_numbers(source_lines: list[str]) -> set[int]:
    """Return 1-indexed line numbers that are *inside* an interface block."""

            continue
        if inside and stripped == "}":
            inside = False
            continue
        if inside:
            result.add(idx)
    return result

### Fix B — Correct the `CursorBounded` bound from `n+1` to `n` (semantic error)
The TLA+ invariant `CursorBounded` is `i ≤ N + 1`, where `i` is the scanner's **internal loop cursor** whose terminal sentinel value is `N + 1`. def _interface_body_line_numbers(source_lines: list[str]) -> set[int]:

### Fix B — Correct the `CursorBounded` bound from `n+1` to `n` (semantic error)
The TLA+ invariant `CursorBounded` is `i ≤ N + 1`, where `i` is the scanner's **internal loop cursor** whose terminal sentinel value is `N + 1`. That bound governs an algorithm-internal variable that `scan_file` does not expose. Skeleton `line_number` fields are **file line positions**, constrained by `LineNumberCorrect` to `[1..N]`. Asserting `line_number ≤ N + 1` is therefore one unit weaker than the correct bound and would silently pass a skeleton pointing to `line_number = N + 1` — an off-the-end position that `LineNumberCorrect` would reject. Changed to `≤ n` in both `_verify_cursor_bounded` and the inline assertion in `TestCursorBoundedInvariant.test_topology_canonical_7_lines`.

The empty-file case was the only test that called no invariant helpers. All four invariants hold vacuously on an empty skeleton list; calling `_all_invariants` here is free and closes a gap where a bug in a helper that manifests only on empty input would otherwise go undetected.
