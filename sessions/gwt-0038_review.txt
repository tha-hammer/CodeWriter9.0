## Review Findings

Three semantic issues were found. Here is the analysis followed by the corrected plan.

---

### Issues Identified

**Issue 1 — Critical / Docstring Lie in `test_edge_target_subdir_of_engine_no_cross_contamination`**

When `target = engine / "target_subdir"`, the state root resolves to `engine/target_subdir/.cw9`. Every directory that must satisfy `~UnderEngineRoot(·)` — namely `schema_dir`, `spec_dir`, and `artifact_dir` — lives under `target/.cw9`, which is itself under `engine_root`. Therefore `_under_engine_root(ctx.schema_dir, engine)` returns `True`, and `_inv_no_cross_contamination` **fails** in this topology. The test body correctly avoids calling it, but the docstring falsely claims *"all invariants hold"*.

**Issue 2 — Moderate / `BoundedExecution` silently absent from `_verify_all_invariants`**

The `AllInvariants` bridge verifier explicitly lists `BoundedExecution` as one of its twelve conditions. `_verify_all_invariants` only calls eleven helpers. Because `step_count` is not an observable attribute of the Python API, the invariant cannot be asserted directly, but the omission should be documented with a comment so future maintainers do not assume it was forgotten.

**Issue 3 — Minor / Incomplete coverage in `test_edge_engine_subdir_of_target_no_cross_contamination`**

When `engine = target / "engine_subdir"`, all eleven testable invariants hold (the directories are disjoint from both roots' perspectives). The test only asserts four of them. Calling `_verify_all_invariants` here closes the gap without risk of false failure.

    """UnderStateRoot(p) == p.base = TargetRoot /\ p.under_cw9 = TRUE"""
    state_root = target_root / ".cw9"
    try:

        return True
    except ValueError:

    """UnderEngineRoot(p) == p.base = EngineRoot"""
