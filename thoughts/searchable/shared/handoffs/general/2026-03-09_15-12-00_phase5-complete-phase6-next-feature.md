## Learnings

### LLM consistently hits these PlusCal pitfalls
- **Labels inside `with`:** PlusCal forbids labels inside `with` blocks. The prompt must explicitly warn against this.
- **Double assignment in `with`:** Can't assign to the same variable twice in one `with` block.
- **Infinite `Seq()` in invariants:** TLC cannot evaluate `\E path \in Seq(S)` — must use bounded `[1..N -> S]` instead.
- **LeafHasNoImpact invariant:** LLMs consistently get this wrong on first attempt (operator precedence issue with `/\` vs `=>`). The retry mechanism handles it.

### Pipeline convergence pattern
- Attempt 1 typically compiles but violates an invariant
- Retry with counterexample feedback fixes it on attempt 2
- The 5-retry limit in `run_impact_loop.py` provides enough headroom

### Bridge needs compiled TLA+ not raw PlusCal
- `run_bridge()` requires the compiled TLA+ output (after `pcal.trans`), not the raw LLM response
- `LoopStatus.compiled_spec_path` now exposes the compiled file path
- The loop script reads from this path for bridge input

### Test generation must be mechanical
- Tests must be generated by code from bridge artifacts, not hand-written from reading JSON
- `generate_tests()` in `run_impact_loop.py` maps bridge verifiers → pytest methods
- TLA+ backslashes (`\A`, `\in`, `\/`) must be sanitized when embedding in Python strings

### claude-agent-sdk usage (unchanged from Phase 4)
- Package: `pip install claude-agent-sdk`
- Must `os.environ.pop("CLAUDECODE", None)` before importing
- `allowed_tools=[]`, `max_turns=1` for pure text generation
- Model: `claude-sonnet-4-20250514`

### Ground rule reminder
**The LLM generates specs, not the implementation agent.** If the LLM-generated spec fails TLC, send the error back through the retry loop. Never hand-write or hand-fix specs — that bypasses the pipeline and defeats self-hosting. Similarly, tests must be mechanically generated from bridge artifacts, not hand-written by reading JSON.
