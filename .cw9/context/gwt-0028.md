# Context for gwt-0028: dfs_before_sweep

IMPORTANT: This spec MUST model TWO-PHASE SEQUENCING (DFS phase completes before sweep phase).
Do NOT model rate limiting, semaphores, env vars, or SDK imports.
The MODULE name MUST be "DfsBeforeSweep".

## Behavior
DFS phase iterates entry points sequentially, recursively extracting callees.
Only after ALL entry points are fully DFS-explored does the sweep phase begin.
The sweep processes remaining unextracted records concurrently.

## Key Variables
- phase: "dfs" | "sweep" | "done"
- entry_points: set of starting UUIDs (use CONSTANT EntryPoints)
- visited: set of UUIDs processed by DFS
- pending: set of UUIDs needing sweep extraction (use CONSTANT InitialPending)
- dfs_complete: boolean flag
- sweep_active: boolean flag

## Key Invariants
- PhaseSequencing: phase = "sweep" => dfs_complete = TRUE
- NoSweepDuringDfs: phase = "dfs" => sweep_active = FALSE
- AllEntryPointsVisited: phase != "dfs" => visited = EntryPoints
- BoundedExecution: step_count <= MaxSteps

## PlusCal Constraints — CRITICAL

### One assignment per variable per label
PlusCal forbids assigning the same variable twice within the same label.
BAD (will cause "Multiple assignment to sweep_active"):
```
SweepStep:
    sweep_active := TRUE;
    \* ... do work ...
    sweep_active := FALSE;
```
GOOD (split into separate labels):
```
SweepActivate:
    sweep_active := TRUE;
SweepExtract:
    \* ... do work ...
    sweep_active := FALSE;
```

### Avoid \subseteq on CONSTANTs in invariants
TLC cannot evaluate `visited \subseteq EntryPoints` when EntryPoints is a model value set.
Use equality checks instead: `visited = EntryPoints` (at terminal states) or
membership checks: `\A u \in visited : u \in EntryPoints`.

### Use finite sets in .cfg
The .cfg file must define EntryPoints, InitialPending as small finite sets
(e.g., {e1, e2} and {p1, p2}) and MaxSteps as a small integer (e.g., 10).

### No duplicate MODULE headers
The output must contain exactly ONE `---- MODULE DfsBeforeSweep ----` line
and exactly ONE `====` terminator. Do not repeat the module.

### No \* BEGIN TRANSLATION markers
Do NOT include `\* BEGIN TRANSLATION` / `\* END TRANSLATION` markers.
Do NOT include `Spec ==` or `THEOREM` definitions after the algorithm block.
Let pcal.trans handle the translation. The module should end with:
```
end algorithm; *)

====
```
