# Context for gwt-0079: plan-review-mode-detection

## Behavior
Given neither --self nor --external flag is supplied, when cmd_plan_review() determines the mode, then if .cw9/ directory exists under target_dir mode is 'external' (using EXTERNAL_REVIEWS); otherwise mode is 'self' (using SELF_REVIEWS); explicit flags override auto-detection.

## Concrete Data Shapes

```
Flags == [self_hosting: BOOLEAN, external: BOOLEAN]
Mode == {"self", "external"}
DirExists == BOOLEAN  \* whether .cw9/ exists

ModeResult == [mode: Mode, reviews: {"SELF_REVIEWS", "EXTERNAL_REVIEWS"}]
```

## Key Invariants to Model

- **ExplicitOverride**: if self_hosting=TRUE then mode="self" regardless of DirExists
- **ExplicitOverride2**: if external=TRUE then mode="external" regardless of DirExists
- **AutoDetectExternal**: if neither flag set AND DirExists=TRUE then mode="external"
- **AutoDetectSelf**: if neither flag set AND DirExists=FALSE then mode="self"
- **ModeReviewConsistency**: mode="self" => reviews=SELF_REVIEWS, mode="external" => reviews=EXTERNAL_REVIEWS

## Modeling Approach

- CONSTANTS: SelfFlag (BOOLEAN), ExternalFlag (BOOLEAN), CW9DirExists (BOOLEAN)
- Variables: mode (Mode), reviews_list (review set name)
- Actions: DetermineMode
- Invariants: ExplicitOverride, AutoDetectExternal, AutoDetectSelf, ModeReviewConsistency

## Anti-Patterns (DO NOT USE)

- Do NOT model both flags as TRUE simultaneously — that's undefined behavior (pick one)
- Do NOT model file system operations — just model the boolean existence check
- Keep the state space small — 2 flags x 1 dir-exists = 8 states total
