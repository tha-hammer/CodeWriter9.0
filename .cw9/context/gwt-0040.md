# Context for gwt-0040: Test generation 3-pass pipeline

## Component
`python/registry/test_gen_loop.py` — main pipeline (line 354)

## Behavior
Three strictly sequential LLM passes produce a test file:

**Pass 1 — Plan** (line 354-358): `build_test_plan_prompt(ctx)` → LLM → `test_plan` string
**Pass 2 — Review** (line 360-363): `build_review_prompt(test_plan, ctx)` → LLM → `reviewed_plan` string
**Pass 3 — Codegen** (line 365-371): `build_codegen_prompt(reviewed_plan, ctx)` → LLM → raw response → `extract_code_from_response()` → written to `test_path`

After Pass 3, mechanical verification runs:
1. `compile()` — Python AST parse (SyntaxError = fail)
2. `pytest --collect-only` — import + fixture discovery (30s timeout)
3. `pytest -x -v` — full test execution (120s timeout)

If verification fails, a retry loop (bounded by `max_attempts`) rebuilds from the codegen step with error context.

## Key Invariant
Passes are strictly sequential — Pass 2 requires Pass 1 output, Pass 3 requires Pass 2 output. Verification never runs until Pass 3 writes the file. The retry loop only retries codegen (Pass 3), not the plan/review passes.

## State Variables for TLA+ Model
- `pass_completed`: {NONE, PLAN, REVIEW, CODEGEN}
- `verification_stage`: {NONE, COMPILE, COLLECT, RUN}
- `verification_passed`: boolean
- `attempt`: 1..max_attempts
- `file_written`: boolean

## Terminal Condition
Either verification passes (all 3 stages) or max_attempts exhausted.
