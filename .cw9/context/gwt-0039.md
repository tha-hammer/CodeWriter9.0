# Context for gwt-0039: Loop retry prompt with classified error

## Component
`python/registry/loop_runner.py` — `build_retry_prompt()` (line 169)

## Behavior
When a TLC verification fails, the error is classified into one of 8 `TLCErrorClass` values. `build_retry_prompt` assembles a retry prompt that includes:

1. The complete `initial_prompt` (original GWT + context)
2. A `## RETRY` section header with attempt number
3. Classified error instruction block from `_ERROR_CLASS_INSTRUCTIONS[error_class]` (if class has an entry)
4. Previous LLM output verbatim
5. Error message from TLC
6. Counterexample details (violated invariant, summary, state trace) — conditional on presence
7. TLC output
8. Closing instruction

## Error Classes with Instructions
- SYNTAX_ERROR: PlusCal translation failure
- PARSE_ERROR: TLA+ parse/semantic error
- TYPE_ERROR: TLC runtime type mismatch
- INVARIANT_VIOLATION: counterexample found
- DEADLOCK: reachable state with no enabled actions
- CONSTANT_MISMATCH: missing CONSTANT/operator declaration

TIMEOUT and UNKNOWN have no instruction entries — classification section is omitted.

## Key Invariant
Every retry prompt starts with the full initial_prompt. Classification is best-effort: missing/unrecognized class produces no classification section but the prompt is still valid.

## State Variables for TLA+ Model
- `error_class`: enum of 8 values + None
- `has_instruction`: boolean — whether error_class maps to an instruction block
- `has_counterexample`: boolean — whether status.counterexample is non-None
- `prompt_sections`: set of sections included in output

## Terminal Condition
The assembled prompt contains initial_prompt + retry header + (optional classification) + previous output + error details.
