# Context for gwt-0084: orchestrate_reviews_with_correction_init

## Behavior
Given --auto-fix set and --max-retries not provided, when `_orchestrate_reviews_with_correction()` initialises retry budget, then max_retries defaults to 2 (3 total attempts per pass: 1 initial + 2 retries).

## Concrete Data Shapes

```
CorrectionState == [
    max_retries: NAT,          \* default 2
    attempt_counts: DICT(STRING -> NAT),  \* per-pass attempt counter
    total_attempts_limit: NAT, \* max_retries + 1 (initial attempt)
]

PASS_DEPENDENCIES = {
    "artifacts": [],
    "coverage": ["artifacts"],
    "interaction": ["artifacts"],
    "abstraction_gap": ["coverage", "interaction"],
    "imports": ["abstraction_gap"],
}

\* _orchestrate_reviews_with_correction signature:
\* async def _orchestrate_reviews_with_correction(
\*     mode, plan_path, cwd, phase, json_mode, max_retries=2
\* ) -> dict
```

## Key Invariants to Model

- **DefaultRetries**: when max_retries parameter not provided, max_retries = 2
- **TotalAttempts**: total attempts per pass = max_retries + 1 (1 initial + max_retries corrections)
- **RetryBudgetPerPass**: each pass has its own independent retry counter
- **InitialState**: all attempt_counts start at 0 before first execution
- **BudgetExhaustion**: when attempt_counts[pass] > max_retries, no more corrections for that pass

## Modeling Approach

- CONSTANTS: MaxRetries (default 2), PassNames (set of pass names)
- Variables: attempt_counts (function from PassName to NAT), budget_exhausted (set of PassName)
- Actions: InitBudget, AttemptPass(name), ExhaustBudget(name)
- Invariants: DefaultRetries, TotalAttempts, BudgetExhaustion

## Anti-Patterns (DO NOT USE)

- Do NOT model the correction pass contents -- just model the retry counter
- Do NOT conflate max_retries with total attempts -- max_retries=2 means 3 total attempts
- Do NOT model global retry budget -- each pass has independent retry tracking
- Do NOT model the dependency graph here -- that is existing gwt-0077's concern
- Do NOT use unbounded naturals for retry counts -- max_retries is small (0..5 range)
