# Context for gwt-0065: seam_satisfied_no_report

IMPORTANT: This spec MUST model the ABSENCE of a mismatch when types are compatible.
Do NOT model mismatch detection, unresolved edges, or the full report.
The behavior is about CONFIRMING TYPE COMPATIBILITY at a single seam.

## Behavior
Given crawl.db has caller A with outs [{ok, "list[FnRecord]"}] and callee B with
ins [{name: "records", type_str: "list[FnRecord]", source: "internal_call", source_uuid: A.uuid}],
when check_seam(A.uuid, B.uuid) is called,
then no SeamMismatch is reported because the type contract is satisfied via exact match.

## Concrete Data Shapes

```python
from dataclasses import dataclass
from registry.crawl_types import DispatchKind

@dataclass
class SeamMismatch:
    caller_uuid: str
    callee_uuid: str
    caller_function: str
    callee_function: str
    callee_input_name: str
    expected_type: str
    provided_type: str
    dispatch: DispatchKind
    severity: str            # "type_mismatch" | "unresolved" | "no_ok_output"
```

When check_seam returns an empty list, no mismatch was found — the seam is satisfied.

## Key Invariants to Model
- NoFalsePositive: if caller's ok output type_str IS compatible with callee's input type_str, the result list MUST be empty
- CompatibilityTiers: exact match ("list[FnRecord]" == "list[FnRecord]") is tier 1. Normalized match (case insensitive, Dict==dict) is tier 2. Structural subtype (str satisfies str|None) is tier 3. Any tier satisfying → empty result.
- Reflexivity: type_compatible(T, T) = TRUE for all T

## Modeling Approach
- CONSTANTS: TypeValues (set of type strings), CompatPairs (set of (provided, expected) pairs that are compatible — includes all reflexive pairs)
- Variables: caller_outs, callee_ins, result (set of mismatches), phase
- Actions: CheckSeam, ConfirmCompatible (no mismatch added)
- Invariants: NoFalsePositives (if (provided, expected) in CompatPairs then result = {}), Reflexive (for all T: (T, T) in CompatPairs)

## Test Interface (MANDATORY)

```python
from registry.seam_checker import check_seam
from registry.crawl_types import FnRecord, InField, OutField, InSource, OutKind, DispatchKind

# Exact match case
caller = FnRecord(
    uuid="aaaa-1111", function_name="get_records", file_path="a.py",
    src_hash="abc", ins=[], do_description="returns records",
    outs=[OutField(name=OutKind.OK, type_str="list[FnRecord]")]
)
callee = FnRecord(
    uuid="bbbb-2222", function_name="process_records", file_path="b.py",
    src_hash="def", ins=[InField(
        name="records", type_str="list[FnRecord]", source=InSource.INTERNAL_CALL,
        source_uuid="aaaa-1111"
    )], do_description="processes records", outs=[]
)

mismatches = check_seam(caller, callee)
assert len(mismatches) == 0  # No mismatch — types are compatible

# Also test normalized match: str satisfies str | None
caller2 = FnRecord(
    uuid="cccc-3333", function_name="get_name", file_path="c.py",
    src_hash="ghi", ins=[], do_description="returns name",
    outs=[OutField(name=OutKind.OK, type_str="str")]
)
callee2 = FnRecord(
    uuid="dddd-4444", function_name="use_name", file_path="d.py",
    src_hash="jkl", ins=[InField(
        name="name", type_str="str | None", source=InSource.INTERNAL_CALL,
        source_uuid="cccc-3333"
    )], do_description="uses name", outs=[]
)
mismatches2 = check_seam(caller2, callee2)
assert len(mismatches2) == 0  # str satisfies str | None (narrower into wider union)
```

## Anti-Patterns (DO NOT USE)
- Do NOT model mismatch detection (that's gwt-0064)
- Do NOT model the type_compatible function internals — just model the compatibility relation as a set of pairs
- Do NOT test against CrawlStore — check_seam operates on FnRecord objects directly
- Do NOT add false-positive mismatches to test the absence — just verify empty result
