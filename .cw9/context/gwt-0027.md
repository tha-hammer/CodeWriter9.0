# Context for gwt-0027: schema_init_idempotent

IMPORTANT: This spec MUST model SCHEMA INITIALIZATION IDEMPOTENCY.
Do NOT model semaphores, workers, rate limiting, concurrent extraction, or API calls.
The behavior is about running DDL statements (CREATE TABLE IF NOT EXISTS) multiple times
and verifying that existing data is preserved. The MODULE name MUST be "SchemaInitIdempotent".

## Behavior
Given a crawl.db that was previously initialized with the full schema,
when connect() is called again (executescript runs CREATE TABLE IF NOT EXISTS),
then the existing tables, indexes, and data are preserved unchanged.

## Implementation
CrawlStore.connect() calls:
```python
self._conn = sqlite3.connect(str(self.db_path))
self._conn.row_factory = sqlite3.Row
self._conn.executescript(_SCHEMA_SQL)
```

_SCHEMA_SQL uses IF NOT EXISTS for every statement:
- CREATE TABLE IF NOT EXISTS records (...)
- CREATE TABLE IF NOT EXISTS ins (...)
- CREATE TABLE IF NOT EXISTS outs (...)
- CREATE TABLE IF NOT EXISTS maps (...)
- CREATE TABLE IF NOT EXISTS test_refs (...)
- CREATE TABLE IF NOT EXISTS entry_points (...)
- CREATE TABLE IF NOT EXISTS crawl_runs (...)
- CREATE VIEW IF NOT EXISTS deps AS ...
- CREATE VIEW IF NOT EXISTS stale_candidates AS ...
- CREATE INDEX IF NOT EXISTS idx_... ON ...

PRAGMAs applied every connection:
- PRAGMA journal_mode = WAL
- PRAGMA foreign_keys = ON

## Key Invariants to Model
- Idempotency: running _SCHEMA_SQL N times produces the same schema as running it once
- Data preservation: existing rows in all tables are unchanged after re-init
- No DDL errors: IF NOT EXISTS prevents "table already exists" errors
- PRAGMA safety: WAL and foreign_keys are session-level settings, re-applied each connection

## Modeling Approach
- CONSTANTS: Tables (set of table names), MaxReconnects
- Variables: schema_exists (set of tables that exist), data (table -> set of rows), connected (boolean)
- Actions: Connect (apply PRAGMAs + executescript), Disconnect, InsertRow, Reconnect
- Invariants: SchemaStable (schema_exists after connect == schema_exists after reconnect), DataPreserved (data before reconnect == data after reconnect), NoPragmaLoss (WAL and FK always on when connected)
