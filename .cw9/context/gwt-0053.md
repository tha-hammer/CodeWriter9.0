# Context for gwt-0053: scanner_rust_nested_depth

## Behavior
Given a Rust source file with fn declarations inside impl blocks,
when scan_file is called,
then methods inside impl blocks have class_name set to the impl type,
and free functions have class_name=None. Trait methods are skipped.

Rust nesting: IMPL BLOCK TRACKING via brace depth + is_trait flag.
