# Context for gwt-0034: pipeline_step_ordering

IMPORTANT: This spec models the THREE-PHASE PIPELINE ORDERING — setup before loop, loop before
bridge, with early exit on setup failure. The MODULE name MUST be "PipelineStepOrdering".

## Behavior
`cmd_pipeline` runs three phases in strict order:
1. **Setup**: init + extract. If either fails, return immediately (early exit).
2. **Loop**: For each GWT, run verification. Failures do NOT short-circuit — all GWTs are attempted. Results tracked in loop_results map.
3. **Bridge**: Only processes GWTs that passed the loop phase. Failures do NOT short-circuit.

Final exit code: 0 if all steps passed, 1 if any failed.

## Key Variables
- phase: "setup" | "loop" | "bridge" | "done"
- setup_result: "pending" | "pass" | "fail"
- gwt_ids: set of GWT IDs to process (use CONSTANT GwtIds)
- loop_done: set of GWT IDs that completed loop (pass or fail)
- loop_passed: set of GWT IDs that passed loop verification
- bridge_done: set of GWT IDs that completed bridge
- exit_code: 0 | 1

## Key Invariants
- SetupBeforeLoop: phase = "loop" => setup_result = "pass"
- LoopBeforeBridge: phase = "bridge" => loop_done = GwtIds
- BridgeOnlyVerified: \A g \in bridge_done : g \in loop_passed
- EarlyExitOnSetupFail: setup_result = "fail" => phase = "done"
- AllGwtsAttempted: phase # "loop" /\ setup_result = "pass" => loop_done = GwtIds
- ExitCodeCorrect: phase = "done" /\ setup_result = "pass" => (exit_code = 0 <=> loop_passed = GwtIds /\ bridge_done = loop_passed)
- BoundedExecution: step_count <= MaxSteps

## PlusCal Constraints — CRITICAL

### One assignment per variable per label
PlusCal forbids assigning the same variable twice within the same label.

### Use finite sets in .cfg
GwtIds = {g1, g2}
MaxSteps = 20 (needs more steps for multi-GWT processing)

### No duplicate MODULE headers
Exactly ONE `---- MODULE PipelineStepOrdering ----` and ONE `====`.

### No \* BEGIN TRANSLATION markers
Do NOT include `\* BEGIN TRANSLATION` / `\* END TRANSLATION` markers.

### CRITICAL: Constant classification hint
For any CONSTANT that is used as a set, add `ASSUME X # {}` after the CONSTANTS block.
This tells the TLC cfg generator to treat it as a finite set, not a numeric value.
For this spec, GwtIds needs `ASSUME GwtIds # {}`. MaxSteps is numeric — no ASSUME needed.
