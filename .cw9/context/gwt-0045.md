## Behavior
Phase 3 runs after node creation (Phase 1) and edge creation (Phase 2):

1. `store.get_all_uuids()` fetches ALL UUIDs from records table (including external records)
2. For each node in DAG:
   - Skip if `node.kind != NodeKind.RESOURCE`
   - Skip if `nid in crawl_uuids`
   - Skip if `not _looks_like_uuid(nid)` (must match 8-4-4-4-12 hex format)
   - Otherwise: `dag.remove_node(nid)` — removes node and all incident edges
3. `nodes_removed` counter tracks deletions

## `_looks_like_uuid` check (line 78)
Regex: `^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$`
This ensures `gwt-NNNN` and `req-NNNN` IDs are never treated as orphans.

## Key Invariant
- Only RESOURCE nodes with UUID-format IDs are candidates for removal
- Non-RESOURCE nodes (BEHAVIOR, REQUIREMENT) are NEVER removed
- Non-UUID-format RESOURCE nodes are NEVER removed
- External record UUIDs protect nodes from removal (get_all_uuids includes externals)
- After cleanup: no RESOURCE node with UUID-format ID exists in DAG unless that UUID is in crawl.db
- Edge removal is a side effect of node removal (dag.remove_node handles this)

## State Variables for TLA+ Model

- `dag_resource_nodes`: set of {id, kind} where kind == RESOURCE
- `dag_other_nodes`: set of {id, kind} where kind != RESOURCE
- `uuid_format_ids`: subset of dag_resource_nodes where id matches UUID format
- `orphans`: uuid_format_ids \ crawl_uuids
- `nodes_removed`: counter
