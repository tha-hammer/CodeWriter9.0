# Context for gwt-0070: cross_cutting_cli_integrated

## Behavior
Given cmd_seams() is invoked with --cross-cutting flag and a valid rules file exists,
When cmd_seams() executes,
Then output includes both the standard seam report and a behavioral report section; with --json the output includes a behavioral_violations array; without --cross-cutting the output is unchanged from existing behavior; with --cross-cutting but missing rules file, prints error to stderr and exits non-zero.

## Key Invariants to Model

- BackwardCompatible: no --cross-cutting flag → output identical to pre-existing behavior
- FlagEnablesBehavioral: --cross-cutting → behavioral report appended to output
- JsonIncludesBehavioral: --json + --cross-cutting → JSON has behavioral key
- MissingRulesError: --cross-cutting + no rules file → stderr error, exit 1
- BothReportsPresent: --cross-cutting → both seam report AND behavioral report in output

## Modeling Approach

- CONSTANTS: Flags = {cross_cutting, json, verbose}, RulesFileExists (BOOLEAN)
- Variables: flags_set (subset of Flags), stdout, stderr, exit_code, phase
- Actions: ParseFlags, LoadRules (conditional on cross_cutting flag), RunSeamCheck, RunBehavioralCheck (conditional), FormatOutput, Exit
- Invariants: BackwardCompatible, MissingRulesError, BothReportsPresent

import argparse
from io import StringIO
from unittest.mock import patch

# Setup: create a target dir with .cw9/crawl.db and schema/cross_cutting_rules.json

with patch("sys.stdout", new_callable=StringIO) as mock_stdout:
    exit_code = cmd_seams(args)
assert "behavioral" not in mock_stdout.getvalue().lower()

     patch("sys.stdout", new_callable=StringIO) as mock_stdout:
    exit_code = cmd_seams(args)
assert exit_code == 1
assert "error" in mock_stderr.getvalue().lower()
assert "behavioral" not in mock_stdout.getvalue().lower()

# Test 4: --json --cross-cutting → JSON includes behavioral key
args.cross_cutting = True
args.output_json = True
# (restore rules file)

assert "violations" in output["behavioral"]
