# Context for gwt-0074: JavaScript/TypeScript Entry Point Discovery

## Behavior
Given a JavaScript or TypeScript project root with skeletons from the scanner and source files on disk,
when the JS/TS discoverer runs,
then it returns EntryPoint objects for:
- Express/Fastify/Koa route handlers (entry_type=HTTP_ROUTE with route and method)
- Exported functions from index files (entry_type=PUBLIC_API for library codebase type)
- bin targets from package.json (entry_type=CLI_COMMAND)
- Commander/yargs CLI command registrations (entry_type=CLI_COMMAND)

node_modules, .min.js, and .d.ts files are excluded. Missing package.json does not cause errors.

## Concrete Data Shapes

```python
@dataclass
class Skeleton:
    function_name: str
    file_path: str
    line_number: int
    class_name: str | None = None
    visibility: str = "public"  # "public" means `export` keyword present
    is_async: bool = False
    params: list = field(default_factory=list)
    return_type: str | None = None
    file_hash: str = ""

@dataclass
class EntryPoint:
    file_path: str
    function_name: str
    entry_type: EntryType  # HTTP_ROUTE, CLI_COMMAND, PUBLIC_API, MAIN
    route: str | None = None
    method: str | None = None
```

## Scanner Signals for JS/TS

| Signal | JavaScript | TypeScript |
|---|---|---|
| `visibility="public"` | `export` keyword, `module.exports` | `export` keyword |
| `class_name=None` | top-level function | top-level function |
| `is_async` | yes | yes |
| `return_type` | always None | available |

## Framework Patterns to Detect

### Express/Koa/Hapi Routes (file scan)
```javascript
app.get("/users", getUsers)
app.post("/users", createUser)
router.put("/users/:id", updateUser)
app.delete("/users/:id", deleteUser)
```
Regex: `\.\s*(get|post|put|delete|patch|head|options)\s*\(\s*["']([^"']+)["']`

### package.json bin field (manifest parse)
```json
{"bin": "./cli.js"}
{"bin": {"mycli": "./bin/cli.js", "helper": "./bin/help.js"}}
```

### package.json main field (manifest parse)
```json
{"main": "index.js"}
{"main": "dist/index.js"}
```

### Commander/yargs CLI commands (file scan)
```javascript
program.command("build")
program.command("serve")
```
Regex: `\.command\s*\(\s*["'](\w+)["']`

## Key Invariants to Model

- **NodeModulesExcluded**: No entry point has a file_path containing "node_modules"
- **MinJsExcluded**: No entry point has a file_path ending in ".min.js"
- **DtsExcluded**: No .d.ts files produce route or CLI entry points
- **PackageJsonOptional**: Missing package.json returns empty manifest results (no error)
- **PackageJsonMalformed**: Invalid JSON in package.json returns empty manifest results (no error)
- **LibraryPublicApi**: When codebase_type="library", public top-level skeletons produce PUBLIC_API entries
- **DeduplicatedResults**: No two entry points share the same (file_path, function_name) pair
- **RouteMethodUppercase**: HTTP route methods are stored as uppercase strings ("GET", "POST", etc.)
- **FallbackToExports**: When no entry points found (not library, no routes, no CLI, no manifest), all public top-level skeleton exports become PUBLIC_API entries

## Test Interface (MANDATORY)

```python
from registry.entry_points import discover_entry_points
from registry.crawl_types import EntryPoint, EntryType, Skeleton
from pathlib import Path
import json

# Helper to build test skeletons
def _skel(file_path, function_name, *, visibility="public", class_name=None):
    return Skeleton(
        function_name=function_name,
        file_path=file_path,
        line_number=1,
        visibility=visibility,
        class_name=class_name,
    )

# Express route detection
(tmp_path / "app.js").write_text('app.get("/users", getUsers)\napp.post("/users", createUser)\n')
eps = discover_entry_points(tmp_path, "web_app", lang="javascript", skeletons=[])
assert any(ep.route == "/users" and ep.method == "GET" for ep in eps)

# package.json bin field
(tmp_path / "package.json").write_text(json.dumps({"bin": {"mycli": "./cli.js"}}))
eps = discover_entry_points(tmp_path, "cli", lang="javascript", skeletons=[])
assert any(ep.function_name == "mycli" and ep.entry_type == EntryType.CLI_COMMAND for ep in eps)

# Public API from skeletons (library type)
skels = [_skel("index.js", "createApp", visibility="public")]
eps = discover_entry_points(tmp_path, "library", lang="javascript", skeletons=skels)
assert any(ep.function_name == "createApp" and ep.entry_type == EntryType.PUBLIC_API for ep in eps)

# node_modules exclusion (tested via file scan, not skeletons)
nm = tmp_path / "node_modules" / "express"
nm.mkdir(parents=True)
(nm / "index.js").write_text('app.get("/internal", handler)\n')
eps = discover_entry_points(tmp_path, "web_app", lang="javascript", skeletons=[])
assert not any(ep.route == "/internal" for ep in eps)
```

## Anti-Patterns (DO NOT USE)

- Do NOT scan .ts files when lang="javascript" or vice versa — use lang to select extensions
- Do NOT treat `module.exports` as a route — it's a library export, not an HTTP handler
- Do NOT regex-match HTTP methods inside comments or string literals (acceptable false positive rate)
- Do NOT require package.json to exist — many JS projects don't have one at project root
- Do NOT duplicate the _JS_COMMANDER_RE and _JS_YARGS_RE — they're the same pattern, use one constant

## TLA+ Modeling Hints

- Do NOT use CONSTANTS for boolean flags like HasPackageJson or PkgJsonValid — TLC treats untyped CONSTANTS as model values, not booleans, so `~HasPackageJson` will fail with "~ applied to non-boolean"
- Instead, use variables initialized nondeterministically: `has_pkg_json \in BOOLEAN` in the Init predicate
- Similarly for CodebaseType: use `codebase_type \in {"web_app", "cli", "library", "event_driven"}` as a variable, not a CONSTANT
- Keep the model small: 2-3 skeleton archetypes, 2-3 codebase types, boolean flags for manifest/routes presence
