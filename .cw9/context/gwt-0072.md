# Context for gwt-0072: template_selected_by_annotation

## Behavior
Given a GWT node with metadata field template set to 'observability_state_machine',
When run_loop() processes that GWT node,
Then loads observability_state_machine.tla instead of state_machine.tla for prompt construction; falls back to state_machine.tla when template field is absent; raises FileNotFoundError for unknown template names.

## Concrete Data Shapes

```python
# DAG node structure (from dag.py)
@dataclass
class Node:
    id: str
    kind: NodeKind
    name: str
    given: str = ""
    when: str = ""
    then: str = ""
    path: str | None = None
    description: str = ""
    metadata: dict | None = None  # NEW: {"template": "observability_state_machine"}
```

Template resolution logic (new in loop_runner.py):
```python
# In run_loop(), replace hardcoded template selection:
template_name = "state_machine"  # default
if gwt_node.metadata and "template" in gwt_node.metadata:
    template_name = gwt_node.metadata["template"]

template_path = ctx.template_dir / f"{template_name}.tla"
if not template_path.exists():
    raise FileNotFoundError(f"Template not found: {template_path}")
template_text = template_path.read_text()
```

## Key Invariants to Model

- DefaultFallback: no metadata template field → loads state_machine.tla
- AnnotationRespected: metadata.template = X → loads X.tla
- UnknownTemplateError: metadata.template = nonexistent → FileNotFoundError raised
- NoSilentFallback: unknown template MUST NOT silently fall back to default
- PromptContainsTemplate: _build_prompt output contains the selected template's content

## Modeling Approach

- CONSTANTS: Templates = {"state_machine", "observability_state_machine"}, ExistingFiles (subset of Templates)
- Variables: gwt_metadata (record or NULL), selected_template, template_text, error, phase
- Actions: ReadMetadata, ResolveTemplate (default or from metadata), CheckExists, LoadContent, BuildPrompt
- Invariants: DefaultFallback, AnnotationRespected, UnknownTemplateError, NoSilentFallback

## Anti-Patterns (DO NOT USE)

- Do NOT model the full run_loop() flow — focus on template resolution only
- Do NOT model LLM interaction — template selection happens before the LLM call
- Do NOT model file content — only model whether the file exists and which file is selected
- Do NOT use TLA+ strings for file paths — use symbolic constants for template names
