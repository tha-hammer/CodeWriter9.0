# Context for gwt-0068: cross_cutting_rules_loaded

## Behavior
Given a JSON file at the configured rules path containing an array of rule objects, each with resource_type (string), required_outs (array of strings), and position (one of pre/post/wrap),
When load_cross_cutting_rules(path) is called,
Then returns a list of CrossCuttingRule dataclasses with fields populated from JSON; raises ValueError on missing fields, invalid position values, or malformed JSON; raises FileNotFoundError if file absent.

## Concrete Data Shapes

```python
@dataclass
class CrossCuttingRule:
    resource_type: str          # e.g., "database", "external_api", "cache"
    required_outs: list[str]    # e.g., ["audit_event", "span_id"]
    position: str               # "pre" | "post" | "wrap"
```

JSON input shape:
```json
{
  "rules": [
    {
      "resource_type": "database",
      "required_outs": ["audit_event"],
      "position": "wrap"
    }
  ]
}
```

## Key Invariants to Model

- ValidPosition: position is always one of {"pre", "post", "wrap"}
- CompleteFields: every returned rule has all three fields populated
- FileAbsentError: missing file path → FileNotFoundError, not empty list
- InvalidSchemaError: malformed JSON or missing fields → ValueError, not partial results
- EmptyRulesValid: empty rules array is valid and returns empty list

## Modeling Approach

- CONSTANTS: ValidPositions = {"pre", "post", "wrap"}, RuleFields = {"resource_type", "required_outs", "position"}
- Variables: file_exists (BOOLEAN), json_valid (BOOLEAN), rules_data (set of records), result, error, phase
- Actions: ReadFile (check existence), ParseJSON (check valid), ValidateRules (check fields + positions), Return
- Invariants: NoPartialResults (error ≠ "" ⇒ result = {}), ValidPositionOnly (∀ r ∈ result: r.position ∈ ValidPositions)

## Test Interface (MANDATORY)

```python
from registry.seam_checker import load_cross_cutting_rules
from registry.crawl_types import CrossCuttingRule
from pathlib import Path
import json

# Write valid rules file
rules_path = tmp_path / "cross_cutting_rules.json"
rules_path.write_text(json.dumps({
    "rules": [{"resource_type": "database", "required_outs": ["audit_event"], "position": "wrap"}]
}))

result = load_cross_cutting_rules(rules_path)
assert len(result) == 1
assert isinstance(result[0], CrossCuttingRule)
assert result[0].resource_type == "database"
assert result[0].required_outs == ["audit_event"]
assert result[0].position == "wrap"

# Missing file
import pytest
with pytest.raises(FileNotFoundError):
    load_cross_cutting_rules(Path("/nonexistent/rules.json"))

# Invalid position
bad_path = tmp_path / "bad_rules.json"
bad_path.write_text(json.dumps({"rules": [{"resource_type": "db", "required_outs": [], "position": "INVALID"}]}))
with pytest.raises(ValueError):
    load_cross_cutting_rules(bad_path)
```

## Anti-Patterns (DO NOT USE)

- Do NOT model the JSON parser itself — treat parsing as atomic (succeeds or fails)
- Do NOT use TLA+ sequences for the rules list — use a set of records
- Do NOT model file I/O details — abstract to file_exists boolean
