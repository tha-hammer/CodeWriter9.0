# Context for gwt-0042: CW7 extract produces register-compatible JSON

## Component
`python/registry/cw7.py` — `extract(db_path, session_id=None)` (line 22)

## Behavior
Reads a CW7 SQLite database and produces register-compatible JSON:

1. **Session resolution**: If no session_id given:
   - 0 sessions → ValueError
   - >1 sessions → ValueError (lists available IDs)
   - exactly 1 → auto-selected

2. **Requirements**: Query `requirements WHERE session_id = ?`, map to `{"id": requirement_id, "text": description}`

3. **GWTs**: Query `acceptance_criteria WHERE session_id = ? AND format = 'gwt'`, map to:
   - `criterion_id`: `f"cw7-crit-{id}"` (synthetic namespaced ID)
   - `given`, `when`, `then`: from clause columns (null → "")
   - `parent_req`: requirement_id foreign key
   - `name`: present iff when_clause is non-empty, via `_slugify(when)`

## Output Shape
```json
{
  "requirements": [{"id": "...", "text": "..."}, ...],
  "gwts": [{"criterion_id": "cw7-crit-N", "given": "...", "when": "...", "then": "...", "parent_req": "...", "name"?: "..."}, ...]
}
```

## Key Invariant
- Only `format = 'gwt'` rows produce GWT entries
- `criterion_id` always has the `cw7-crit-` prefix
- `name` field present iff `when_clause` is non-empty
- Session ambiguity is rejected, not resolved silently

## State Variables for TLA+ Model
- `session_count`: 0, 1, or >1
- `session_id_provided`: boolean
- `ac_format`: set of {"gwt", "other"}
- `when_clause_empty`: boolean per AC row
- `output_valid`: boolean — output matches register-compatible shape

## Terminal Condition
Output dict has both "requirements" and "gwts" keys, each a list of correctly-shaped dicts.
