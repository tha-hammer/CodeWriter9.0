# Context for gwt-0075: Go Entry Point Discovery

## Behavior
Given a Go project root with skeletons from the scanner and source files on disk,
when the Go discoverer runs,
then it returns EntryPoint objects for:
- func main() in package main with no receiver (entry_type=MAIN)
- Exported (capitalized) package-level functions with no receiver (entry_type=PUBLIC_API for library codebase type)
- HTTP route registrations via http.HandleFunc, Gin r.GET/POST, Echo e.GET/POST, Chi r.Get/Post (entry_type=HTTP_ROUTE with route and method)
- Cobra command registrations (entry_type=CLI_COMMAND)

_test.go files and dotfiles are excluded.

## Concrete Data Shapes

```python
@dataclass
class Skeleton:
    function_name: str       # e.g., "main", "NewService", "handleHealth"
    file_path: str           # e.g., "cmd/main.go"
    line_number: int
    class_name: str | None = None  # None = package-level func; "MyType" = method receiver
    visibility: str = "public"     # "public" = Capitalized; "private" = lowercase
    is_async: bool = False         # always False for Go
    params: list = field(default_factory=list)
    return_type: str | None = None
    file_hash: str = ""

@dataclass
class EntryPoint:
    file_path: str
    function_name: str
    entry_type: EntryType
    route: str | None = None
    method: str | None = None
```

## Scanner Signals for Go

| Signal | Meaning |
|---|---|
| `visibility="public"` | Function name starts with uppercase letter |
| `visibility="private"` | Function name starts with lowercase letter |
| `class_name=None` | Package-level function (no receiver) |
| `class_name="MyType"` | Method on type MyType (has receiver) |
| `function_name="main"` | Binary entry point (if class_name is None) |

## Framework Patterns to Detect

### Gin/Echo/Chi Routes (file scan)
```go
r.GET("/users", getUsers)
r.POST("/users", createUser)
e.PUT("/users/:id", updateUser)
```
Regex: `\.\s*(GET|POST|PUT|DELETE|PATCH|HEAD|OPTIONS)\s*\(\s*"([^"]+)"`

### net/http HandleFunc (file scan)
```go
http.HandleFunc("/health", healthCheck)
mux.Handle("/api", apiHandler)
```
Regex: `(?:HandleFunc|Handle)\s*\(\s*"([^"]+)"`
Note: HandleFunc is method-agnostic (all HTTP methods). Use method=None, not "GET".

### Cobra CLI Commands (file scan)
```go
var serveCmd = &cobra.Command{
    Use: "serve",
    Run: runServe,
}
```
Regex: `cobra\.Command\s*\{[^}]*Use:\s*"(\w+)"` (with re.DOTALL)

## Key Invariants to Model

- **MainIsTopLevel**: Only `main` functions with `class_name is None` produce MAIN entries
- **MainNotMethod**: `main()` on a receiver (class_name != None) is NOT a MAIN entry point
- **PublicOnlyForLibrary**: PUBLIC_API entries are only produced when codebase_type="library"
- **PublicMeansCapitalized**: Only skeletons with visibility="public" produce PUBLIC_API entries
- **MethodsExcluded**: Skeletons with class_name != None are never PUBLIC_API entries
- **TestFilesExcluded**: Files ending in _test.go are excluded from route/CLI scanning
- **DotfilesExcluded**: Files under directories starting with "." are excluded
- **HandleFuncMethodAgnostic**: http.HandleFunc routes should have method=None (not defaulting to "GET")
- **DeduplicatedResults**: No two entry points share the same (file_path, function_name) pair
- **MainSkipsPublicApi**: A skeleton matching main() uses `continue` — it's not also added as PUBLIC_API

## Test Interface (MANDATORY)

```python
from registry.entry_points import discover_entry_points
from registry.crawl_types import EntryPoint, EntryType, Skeleton
from pathlib import Path

def _skel(file_path, function_name, *, visibility="public", class_name=None):
    return Skeleton(
        function_name=function_name,
        file_path=file_path,
        line_number=1,
        visibility=visibility,
        class_name=class_name,
    )

# main() detection
skels = [_skel("main.go", "main")]
eps = discover_entry_points(tmp_path, "cli", lang="go", skeletons=skels)
assert len(eps) == 1
assert eps[0].entry_type == EntryType.MAIN

# Method named main excluded
skels = [_skel("foo.go", "main", class_name="Server")]
eps = discover_entry_points(tmp_path, "cli", lang="go", skeletons=skels)
assert len(eps) == 0

# Public API filtering
skels = [
    _skel("pkg.go", "NewService", visibility="public"),
    _skel("pkg.go", "helperFunc", visibility="private"),
]
eps = discover_entry_points(tmp_path, "library", lang="go", skeletons=skels)
names = [ep.function_name for ep in eps]
assert "NewService" in names
assert "helperFunc" not in names

# Gin route detection (file scan)
(tmp_path / "main.go").write_text('r.GET("/users", getUsers)\nr.POST("/users", createUser)\n')
eps = discover_entry_points(tmp_path, "web_app", lang="go", skeletons=[])
routes = [ep.route for ep in eps if ep.entry_type == EntryType.HTTP_ROUTE]
assert "/users" in routes
```

## Anti-Patterns (DO NOT USE)

- Do NOT default HandleFunc to method="GET" — it handles all methods; use method=None
- Do NOT use re.DOTALL for route regexes (only for Cobra, where struct spans lines)
- Do NOT scan _test.go files for routes or CLI commands
- Do NOT treat methods (class_name != None) as public API even if visibility="public"

## TLA+ Modeling Hints

- Do NOT introduce new CONSTANTS beyond what the harness provides — TLC has no values for them and throws Java exceptions when you try to index into them
- Use variables initialized with `\in` for nondeterministic choices: `codebase_type \in {"web_app", "cli", "library"}`, `has_routes \in BOOLEAN`
- Model skeletons as a count (integer), not as individual records — use `skel_idx \in 1..NumSkeletons` to iterate
- Use `either/or` for nondeterministic skeleton properties (public vs private, has class_name vs None, function named "main" vs not)
- Keep the model small: 2-3 skeletons, 2-3 source files, boolean flags for file exclusion
