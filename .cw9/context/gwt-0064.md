# Context for gwt-0064: seam_mismatch_detected

IMPORTANT: This spec MUST model type checking across a single caller→callee edge.
Do NOT model crawl.db SQL, the full seam report, or unresolved edges.
The behavior is about DETECTING A TYPE MISMATCH between one caller's ok-output and one callee's input.

## Behavior
Given crawl.db has caller A with outs [{ok, "Dict[str, Any]"}] and callee B with
ins [{name: "config", type_str: "Config", source: "internal_call", source_uuid: A.uuid}],
when check_seam(A.uuid, B.uuid) is called,
then a SeamMismatch is reported with severity "type_mismatch" because B expects "Config"
but A provides "Dict[str, Any]".

## Concrete Data Shapes

```python
from dataclasses import dataclass
from registry.crawl_types import DispatchKind, OutKind, InSource

@dataclass
class SeamMismatch:
    caller_uuid: str
    callee_uuid: str
    caller_function: str
    callee_function: str
    callee_input_name: str
    expected_type: str       # callee's InField.type_str
    provided_type: str       # caller's best-match OutField.type_str
    dispatch: DispatchKind
    severity: str            # "type_mismatch" | "unresolved" | "no_ok_output"
```

Caller has `outs: list[OutField]` where each OutField has `.name` (OutKind enum: ok/err/side_effect/mutation) and `.type_str` (string).
Callee has `ins: list[InField]` where each InField has `.name` (str), `.type_str` (str), `.source` (InSource enum), `.source_uuid` (str|None).

For seam checking, we match callee ins where `source == InSource.INTERNAL_CALL` and `source_uuid == caller.uuid` against caller outs where `name == OutKind.OK`.

## Key Invariants to Model
- MismatchDetected: if caller's ok output type_str does not satisfy callee's input type_str via any compatibility tier, a SeamMismatch with severity="type_mismatch" MUST be produced
- CallerOutputMatched: for each callee input linked to the caller, exactly one comparison is made against caller's ok-output
- SeverityCorrect: the severity field must be "type_mismatch" (not "unresolved" or "no_ok_output")

## Modeling Approach
- CONSTANTS: TypeValues (set of type strings, e.g. {"Dict[str, Any]", "Config", "list[FnRecord]", "str", "str | None"}), CompatPairs (set of compatible (provided, expected) pairs)
- Variables: caller_outs, callee_ins, result (set of mismatches), phase (idle/checking/done)
- Actions: CheckSeam (compare caller ok-output against each callee internal_call input), ReportMismatch (add to result set), NoMismatch (skip compatible pair)
- Invariants: AllInputsChecked (every callee input with source_uuid=caller is compared), MismatchCorrect (result contains mismatch iff types not in CompatPairs)

## Test Interface (MANDATORY)

```python
from registry.seam_checker import check_seam
from registry.crawl_types import FnRecord, InField, OutField, InSource, OutKind, DispatchKind

# Build caller with ok output "Dict[str, Any]"
caller = FnRecord(
    uuid="aaaa-1111", function_name="get_config", file_path="a.py",
    src_hash="abc", ins=[], do_description="returns config dict",
    outs=[OutField(name=OutKind.OK, type_str="Dict[str, Any]")]
)

# Build callee with input expecting "Config" from caller
callee = FnRecord(
    uuid="bbbb-2222", function_name="process", file_path="b.py",
    src_hash="def", ins=[InField(
        name="config", type_str="Config", source=InSource.INTERNAL_CALL,
        source_uuid="aaaa-1111"
    )], do_description="processes config", outs=[]
)

mismatches = check_seam(caller, callee)
assert len(mismatches) == 1
assert mismatches[0].severity == "type_mismatch"
assert mismatches[0].expected_type == "Config"
assert mismatches[0].provided_type == "Dict[str, Any]"
```

## Anti-Patterns (DO NOT USE)
- Do NOT model SQL queries or CrawlStore methods — check_seam takes FnRecord objects, not UUIDs
- Do NOT model the 3-tier type compatibility as a complex function — model it as a set of known-compatible pairs (CompatPairs constant)
- Do NOT model the full seam report (that's gwt-0067)
- Do NOT model unresolved edges (that's gwt-0066)
