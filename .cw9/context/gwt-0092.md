# Context for gwt-0092: Retries exhausted = permanent FAIL

## Behavior

Given a FAIL verdict is returned and the current attempt equals max_retries, when the inner retry loop exits, then the pass is permanently failed, no more correction agents are spawned, dependent passes are blocked, and the summary includes stuck_on and findings.

## Concrete Data Shapes

```python
# Pass state after exhausting retries (max_retries=2)
pass_state = {
    "name": "coverage",
    "attempt": 2,                # == max_retries
    "max_retries": 2,
    "verdict": "FAIL",           # permanent
    "history": [
        {"name": "coverage", "verdict": "FAIL", "session_id": "r1", "cost_usd": 0.05, "result_length": 200, "result_text": "FAIL: issue A"},
        {"name": "coverage", "verdict": "FAIL", "session_id": "r2", "cost_usd": 0.06, "result_length": 250, "result_text": "FAIL: issue A persists"},
    ],
    "corrections": [
        {"name": "coverage_correction", "session_id": "c1", "cost_usd": 0.08, "result_length": 400, "result_text": "Attempted fix..."},
    ],
}

# Summary when retries exhausted
summary = {
    "status": "fail",
    "stuck_on": "coverage",                  # str: name of the pass that couldn't be fixed
    "findings": {
        "coverage": {
            "verdict": "FAIL",
            "issues": ["issue A persists"],   # from last attempt's _extract_findings()
        },
    },
    "passes": {
        "artifacts": {"verdict": "PASS", ...},
        "coverage": pass_state,
        "abstraction_gap": {"verdict": "blocked", ...},  # blocked by coverage
        "imports": {"verdict": "blocked", ...},           # transitively blocked
    },
}
```

## Key Invariants to Model

- **NoCorrection_AtMax**: When attempt == max_retries and verdict == "FAIL", _run_correction_pass() is NOT called. The loop exits.
- **PermanentFail**: Once retries are exhausted, the verdict is permanently FAIL. No external mechanism can retry.
- **StuckOnRecorded**: The summary's stuck_on field names the permanently failed pass.
- **FindingsPreserved**: The findings from the last attempt are preserved in the summary for human review.
- **DependentsBlocked**: All passes that depend (directly or transitively) on the failed pass are marked "blocked".
- **CorrectionCount**: len(corrections) == max_retries - 1. The last attempt gets no correction.

## Modeling Approach

Model the retry loop exit condition explicitly:
```
while attempt <= max_retries:
    verdict := runReview()
    if verdict != "FAIL":
        break
    if attempt < max_retries:
        runCorrection(findings)
    attempt := attempt + 1

(* After loop: *)
if verdict == "FAIL":
    markPermanentFail(pass)
    blockDependents(pass)
    summary.stuck_on := pass.name
```

Key state variables:
- `attempt`: Nat (1..max_retries)
- `verdict`: {"PASS", "WARNING", "FAIL"}
- `permanent`: BOOLEAN (set TRUE when loop exits with FAIL)
- `correctionCount`: Nat
- `blockedPasses`: set of pass names

Invariant: `permanent => (verdict = "FAIL" /\ attempt = max_retries)`
Invariant: `permanent => correctionCount = max_retries - 1`

## Anti-Patterns (DO NOT USE)

- Do NOT run correction on the last attempt. When attempt == max_retries, the FAIL is final.
- Do NOT allow retries beyond max_retries through off-by-one errors. The loop must exit cleanly.
- Do NOT clear findings on permanent failure. They must be preserved for the summary.
- Do NOT mark dependents as "FAIL". They are "blocked" -- a distinct status meaning they never ran.
- Do NOT confuse max_retries with total attempts. max_retries=2 means 2 review attempts (with 1 correction in between).
