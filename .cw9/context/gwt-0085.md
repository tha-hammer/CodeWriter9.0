# Context for gwt-0085: cmd_plan_review_parses_cli_arguments (max-retries)

## Behavior
Given --auto-fix set and --max-retries N provided, when cmd_plan_review() parses CLI arguments, then max_retries=N is forwarded to `_orchestrate_reviews_with_correction()`; 0 means single pass (no correction retries); negative values are rejected by argparse.

## Concrete Data Shapes

```
CLIArgs == [
    plan_path: STRING,
    target_dir: STRING,
    self_hosting: BOOLEAN,
    external: BOOLEAN,
    json_output: BOOLEAN,
    phase: {"pre", "post", "all"},
    auto_fix: BOOLEAN,
    max_retries: NAT,        \* 0..N, negative rejected
]

\* --max-retries is only meaningful when --auto-fix is set
\* --max-retries 0 => single review pass, no corrections
\* --max-retries 3 => up to 3 correction attempts per failing pass
```

## Key Invariants to Model

- **MaxRetriesForwarded**: max_retries value from CLI is passed to _orchestrate_reviews_with_correction
- **ZeroMeansSinglePass**: max_retries=0 => no correction passes run, just the initial review
- **NegativeRejected**: max_retries < 0 => argparse rejects with error before cmd_plan_review runs
- **DefaultWhenOmitted**: --max-retries not provided => uses default value of 2
- **IgnoredWithoutAutoFix**: if auto_fix=FALSE, max_retries has no effect (not forwarded)

## Modeling Approach

- CONSTANTS: MaxRetriesInput (integer from CLI), AutoFix (BOOLEAN)
- Variables: parsed_max_retries (NAT), error_raised (BOOLEAN)
- Actions: ParseMaxRetries, ValidateNonNegative, ForwardToOrchestrator
- Invariants: NegativeRejected, ZeroMeansSinglePass, MaxRetriesForwarded

## Anti-Patterns (DO NOT USE)

- Do NOT model the retry loop itself -- just model that the value is parsed and forwarded
- Do NOT model what happens inside _orchestrate_reviews_with_correction -- that is gwt-0084's concern
- Do NOT allow negative values through validation -- argparse type=int with a validator should catch them
- Do NOT make --max-retries required -- it has a default of 2
- Do NOT test --max-retries without --auto-fix -- it is only meaningful in auto-fix mode
