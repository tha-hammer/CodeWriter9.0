# Context for gwt-0052: scanner_rust_line_ranges

## Behavior
Given a Rust source file with fn declarations at known line positions,
when scan_file is called,
then each Skeleton.line_number equals the 1-indexed line of its fn keyword.
Functions inside trait blocks are skipped.

(* --algorithm ScannerRustLineRanges

    skeletons = {},
    in_trait  = FALSE;

                end if;

                i := i + 1;
        end while;

end process;

end algorithm; *)
