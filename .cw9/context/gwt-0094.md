# Context for gwt-0094: Artifacts failure blocks everything

## Behavior

Given the artifacts pass has exhausted retries and is in terminal FAIL, when the orchestrator tries to schedule coverage, interaction, abstraction_gap, and imports, then all four are blocked without running, and the summary has status=fail with stuck_on=artifacts.

## Concrete Data Shapes

```python
PASS_DEPENDENCIES = {
    "artifacts": [],
    "coverage": ["artifacts"],
    "interaction": ["artifacts"],
    "abstraction_gap": ["coverage", "interaction"],
    "imports": ["abstraction_gap"],
}

# Artifacts is the root of the dependency tree. Its failure cascades to everything.

# Summary when artifacts permanently fails
summary = {
    "status": "fail",
    "stuck_on": "artifacts",
    "findings": {
        "artifacts": {
            "verdict": "FAIL",
            "issues": ["Critical: plan has no implementation artifacts"],
        },
    },
    "passes": {
        "artifacts":       {"verdict": "FAIL",    "attempt": 2, "max_retries": 2},
        "coverage":        {"verdict": "blocked"},
        "interaction":     {"verdict": "blocked"},
        "abstraction_gap": {"verdict": "blocked"},
        "imports":         {"verdict": "blocked"},
    },
    "total_cost_usd": 0.23,   # only artifacts attempts + corrections
}
```

## Key Invariants to Model

- **RootBlock**: Artifacts has no dependencies. If it fails, it is the only pass that ran.
- **AllBlocked**: When artifacts is FAIL, all other passes (coverage, interaction, abstraction_gap, imports) are blocked.
- **NoneRun**: _run_review_pass() is called zero times for any non-artifacts pass.
- **StuckOnArtifacts**: summary.stuck_on == "artifacts".
- **CostOnlyArtifacts**: Total cost reflects only artifacts review attempts and corrections. No other pass incurs cost.
- **EarlyExit**: The orchestrator does not need to iterate through all passes to discover they're blocked. Once artifacts fails, the entire pipeline is done.

## Modeling Approach

Model artifacts as the root node of the dependency DAG. All other passes have artifacts in their transitive dependency set (coverage and interaction directly; abstraction_gap and imports transitively). When artifacts terminates with FAIL:

1. The orchestrator marks artifacts as permanently FAIL.
2. For each remaining pass in topological order, _check_dependencies() returns False.
3. Each is marked "blocked" without invocation.
4. The orchestrator composes the summary and exits.

This is a special case of gwt-0093 (transitive blocking) where the root node fails.

Key state variables:
- `artifactsVerdict`: {"PASS", "WARNING", "FAIL"}
- `otherPassesStatus`: function -> all "blocked" when artifactsVerdict == "FAIL"
- `reviewCallCount`: Nat (should equal number of artifacts attempts only)

## Anti-Patterns (DO NOT USE)

- Do NOT attempt to run coverage or interaction after artifacts fails. They must be blocked immediately.
- Do NOT report stuck_on as the last blocked pass (e.g., "imports"). stuck_on is the root cause: "artifacts".
- Do NOT model this as five independent failures. It is one failure (artifacts) that cascades.
- Do NOT accumulate cost for blocked passes. They never run, so they have zero cost.
- Do NOT mark blocked passes as "FAIL". They are "blocked" -- they were never attempted.
