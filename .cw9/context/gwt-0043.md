# Context for gwt-0043: GWT author validates depends_on against crawl.db

## Component
`python/registry/gwt_author.py` — `validate_depends_on(payload, store)` (line 215)

## Behavior
After the LLM authors GWT entries with `depends_on` UUID lists, validation checks each UUID against crawl.db:

1. `store.get_all_uuids()` fetches ALL UUIDs from `records` table (including external/AX cards)
2. For each GWT entry's `depends_on` list:
   - Retain only UUIDs present in `all_uuids`
   - Remove invalid UUIDs in-place
   - Emit warnings (not exceptions) for removed UUIDs
3. Return list of warning strings

## Key Invariant
- Post-validation, every UUID in `depends_on` exists in crawl.db
- Invalid UUIDs are silently removed, not rejected with exceptions
- The validation scope includes external records (not just internal)
- `crawl.db` must exist before any LLM call (FileNotFoundError raised early)

## Pipeline Context
This is part of `run_gwt_author` which has stages:
1. Read research notes
2. Locate crawl.db (raises if absent — LLM is never called)
3. Extract mentions (function names, file paths, directories from research text)
4. Query relevant IN:DO:OUT cards from crawl.db
5. Build LLM prompt with cards (UUIDs visible to LLM)
6. Call LLM
7. Parse JSON response
8. **Validate depends_on** ← this GWT

## State Variables for TLA+ Model
- `all_uuids`: set of strings (from crawl.db)
- `depends_on_input`: set of strings (from LLM response)
- `depends_on_output`: subset of depends_on_input ∩ all_uuids
- `warnings`: count of removed UUIDs
- `exception_raised`: boolean (always false for invalid UUIDs)

## Terminal Condition
`depends_on_output ⊆ all_uuids` — no invalid UUID survives validation.
