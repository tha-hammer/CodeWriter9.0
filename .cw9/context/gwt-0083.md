# Context for gwt-0083: cmd_plan_review_parses_cli_arguments (auto-fix)

## Behavior
Given cw9 plan-review invoked with --auto-fix, when cmd_plan_review() parses CLI arguments, then `_orchestrate_reviews_with_correction()` is called instead of `_orchestrate_reviews()`.

## Concrete Data Shapes

```
CLIArgs == [
    plan_path: STRING,
    target_dir: STRING,
    self_hosting: BOOLEAN,
    external: BOOLEAN,
    json_output: BOOLEAN,
    phase: {"pre", "post", "all"},
    auto_fix: BOOLEAN,       \* TRUE when --auto-fix is passed
    max_retries: NAT,        \* defaults to 2 when --auto-fix is set
]

DispatchTarget == {"orchestrate_reviews", "orchestrate_reviews_with_correction"}

\* _orchestrate_reviews_with_correction signature:
\* async def _orchestrate_reviews_with_correction(
\*     mode, plan_path, cwd, phase, json_mode, max_retries=2
\* ) -> dict
```

## Key Invariants to Model

- **AutoFixDispatch**: auto_fix = TRUE => dispatch target = "orchestrate_reviews_with_correction"
- **MaxRetriesForwarded**: max_retries from CLI args is passed through to _orchestrate_reviews_with_correction
- **MutualExclusion**: exactly one of {_orchestrate_reviews, _orchestrate_reviews_with_correction} is called
- **FlagIsBoolean**: --auto-fix is a store_true flag (no value argument)

## Modeling Approach

- CONSTANTS: AutoFix (TRUE), MaxRetries (NAT)
- Variables: dispatch_target (DispatchTarget), forwarded_max_retries (NAT)
- Actions: ParseArgs, Dispatch
- Invariants: AutoFixDispatch, MaxRetriesForwarded, MutualExclusion
