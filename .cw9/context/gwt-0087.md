# Context for gwt-0087: extract_findings

## Behavior
Given a review pass completed with json_mode=True in auto-fix mode, when `_extract_findings()` is called with the result text, then it returns structured findings with pass name, issues list, and report path; returns empty findings on parse failure (no exception raised).

## Concrete Data Shapes

```
\* Input: raw result_text from _run_review_pass (expected to be JSON when json_mode=True)
ResultText == STRING

\* Output: structured findings dict
Findings == [
    pass_name: STRING,
    issues: SEQUENCE([
        severity: {"critical", "major", "minor", "info"},
        description: STRING,
        location: STRING | NULL,
        suggestion: STRING | NULL,
    ]),
    report_path: STRING | NULL,  \* path to full report file if written
]

\* Empty findings on failure:
EmptyFindings == [
    pass_name: STRING,
    issues: [],
    report_path: NULL,
]

\* _extract_findings signature:
\* def _extract_findings(result_text: str) -> dict
\*   Returns: {"pass_name": str, "issues": list, "report_path": str | None}
```

## Key Invariants to Model

- **StructuredOutput**: valid JSON input => returns Findings with populated issues list
- **GracefulDegradation**: invalid/unparseable input => returns EmptyFindings (never raises)
- **PassNamePreserved**: pass_name in output matches the pass that produced the result
- **IssuesListType**: issues is always a list (empty list on failure, populated on success)
- **NoExceptionLeaks**: _extract_findings never raises exceptions to the caller
- **EmptyStringHandled**: empty string input => returns EmptyFindings

## Modeling Approach

- CONSTANTS: InputValid (BOOLEAN)
- Variables: parsed (BOOLEAN), findings (Findings | EmptyFindings)
- Actions: AttemptParse, ReturnFindings, ReturnEmpty
- Invariants: GracefulDegradation, IssuesListType, NoExceptionLeaks

## Test Interface (MANDATORY)

```python
from registry.cli import _extract_findings

# Test valid JSON input
valid_json = '{"pass_name": "coverage", "issues": [{"severity": "major", "description": "Missing test for edge case", "location": "src/foo.py:42", "suggestion": "Add test for null input"}], "report_path": "/tmp/coverage_report.md"}'
result = _extract_findings(valid_json)
assert result["pass_name"] == "coverage"
assert len(result["issues"]) == 1
assert result["issues"][0]["severity"] == "major"
assert result["report_path"] == "/tmp/coverage_report.md"

# Test empty issues (all pass)
pass_json = '{"pass_name": "artifacts", "issues": [], "report_path": null}'
result = _extract_findings(pass_json)
assert result["pass_name"] == "artifacts"
assert result["issues"] == []
assert result["report_path"] is None

# Test invalid JSON returns empty findings
result = _extract_findings("This is not JSON at all")
assert result["issues"] == []
assert result["report_path"] is None

# Test empty string returns empty findings
result = _extract_findings("")
assert result["issues"] == []

# Test partial JSON (missing fields) returns gracefully
partial = '{"pass_name": "coverage"}'
result = _extract_findings(partial)
assert isinstance(result["issues"], list)

# Test that no exception is raised for any input
import random, string
garbage = "".join(random.choices(string.printable, k=500))
result = _extract_findings(garbage)  # must not raise
assert isinstance(result, dict)
assert "issues" in result
```

## Anti-Patterns (DO NOT USE)

- Do NOT raise exceptions from _extract_findings -- always return a dict (graceful degradation)
- Do NOT assume result_text is always valid JSON -- LLM output can be malformed
- Do NOT try to re-parse non-JSON text (e.g., regex extraction) -- if it is not JSON, return empty
- Do NOT model the severity levels as an exhaustive enum in TLA+ -- just model "has issues" vs "no issues"
- Do NOT couple this to _run_review_pass -- _extract_findings is a pure function on strings
- Do NOT embed pass_name logic inside _extract_findings -- the pass_name comes from the JSON content itself, or defaults to empty/unknown on parse failure
