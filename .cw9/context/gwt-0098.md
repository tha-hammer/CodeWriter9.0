# Context for gwt-0098: aggregate-verdicts-with-blocked

## Behavior
Given all review passes have reached terminal verdicts and one or more passes have verdict "blocked", when `_aggregate_verdicts()` is called, then overall is "fail" if any pass is FAIL or blocked; the summary separates passed, failed, and blocked buckets; blocked passes do NOT generate new stuck_on entries (they are downstream consequences, not root causes).

## Concrete Data Shapes

```
Verdict == {"pass", "fail", "warning", "blocked"}
PassName == STRING

PassResult == [name: PassName, verdict: Verdict]

AggregationInput == SEQUENCE(PassResult)

AggregationOutput == [
    overall: {"pass", "fail", "warning"},
    passed: SET(PassName),
    failed: SET(PassName),
    blocked: SET(PassName),
    stuck_on: SET(PassName),  \* only from FAIL, never from blocked
]
```

## Key Invariants to Model

- **BlockedImpliesFail**: EXISTS p IN passes : p.verdict = "blocked" => overall = "fail"
- **FailImpliesFail**: EXISTS p IN passes : p.verdict = "fail" => overall = "fail"
- **BucketPartition**: passed UNION failed UNION blocked = {p.name : p IN passes} (every pass in exactly one bucket)
- **BlockedNotStuckOn**: p.verdict = "blocked" => p.name NOT IN stuck_on
- **FailIsStuckOn**: p.verdict = "fail" => p.name IN stuck_on
- **WarningInPassed**: p.verdict = "warning" => p.name IN passed (warnings go to the passed bucket)

## Modeling Approach

- CONSTANTS: NumPasses (2..5), VerdictOptions (Verdict)
- Variables: pass_verdicts (sequence of PassResult), aggregation (AggregationOutput)
- Actions: AssignVerdicts, Aggregate
- Invariants: BlockedImpliesFail, BucketPartition, BlockedNotStuckOn, FailIsStuckOn

## Test Interface (MANDATORY)

```python
from registry.cli import _aggregate_verdicts

# Test: one blocked, rest pass -> overall=fail, blocked bucket populated, no stuck_on for blocked
results = {
    "artifacts": {"verdict": "pass"},
    "coverage": {"verdict": "pass"},
    "interaction": {"verdict": "fail"},
    "abstraction_gap": {"verdict": "blocked"},
    "imports": {"verdict": "blocked"},
}
agg = _aggregate_verdicts(results)
assert agg["overall"] == "fail"
assert "abstraction_gap" in agg["blocked"]
assert "imports" in agg["blocked"]
assert "interaction" in agg["failed"]
assert "abstraction_gap" not in agg.get("stuck_on", [])
assert "imports" not in agg.get("stuck_on", [])
assert "interaction" in agg.get("stuck_on", [])

# Test: all pass, none blocked -> overall=pass
results = {
    "artifacts": {"verdict": "pass"},
    "coverage": {"verdict": "pass"},
    "interaction": {"verdict": "pass"},
    "abstraction_gap": {"verdict": "pass"},
    "imports": {"verdict": "pass"},
}
agg = _aggregate_verdicts(results)
assert agg["overall"] == "pass"
assert len(agg.get("blocked", [])) == 0

# Test: warning + blocked -> overall=fail (blocked dominates warning)
results = {
    "artifacts": {"verdict": "warning"},
    "coverage": {"verdict": "blocked"},
}
agg = _aggregate_verdicts(results)
assert agg["overall"] == "fail"
```

## Anti-Patterns (DO NOT USE)

- Do NOT generate stuck_on entries for blocked passes -- blocked is a consequence, not a root cause
- Do NOT put warnings in the failed bucket -- warnings go to the passed bucket
- Do NOT model the dependency graph here -- aggregation only sees terminal verdicts, not why a pass is blocked
- Do NOT use a separate "blocked" overall status -- blocked passes contribute to overall="fail"
