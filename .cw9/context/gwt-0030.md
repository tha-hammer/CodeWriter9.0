# Context for gwt-0030: incremental_skip_unchanged

IMPORTANT: This spec MUST model HASH-BASED INCREMENTAL SKIP logic.
Do NOT model concurrent workers, semaphores, signal handling, or SDK imports.
The MODULE name MUST be "IncrementalSkip".

## Behavior
In incremental mode, _should_skip(uuid) checks:
1. Is incremental mode enabled? If not, never skip.
2. Is the record's do_description a terminal state (not SKELETON_ONLY/EXTRACTION_FAILED)? If not terminal, don't skip.
3. Does the stored src_hash match the current file's hash? If yes, skip.

extract_fn is NEVER called for skipped records.

## Key Variables
- incremental: boolean (TRUE or FALSE, nondeterministic initial choice)
- records: map of uuid -> {do_description, src_hash}
- current_hashes: map of uuid -> current file hash
- skipped: set of UUIDs that were skipped
- extracted: set of UUIDs that were extracted
- remaining: set of UUIDs still to process

## Key Invariants
- SkipOnlyWhenIncremental: skipped != {} => incremental = TRUE
- SkipOnlyWhenHashMatch: forall u in skipped: records[u].src_hash = current_hashes[u]
- SkipNeverExtracted: skipped and extracted are disjoint
- UnchangedNeverExtracted: incremental /\ terminal /\ hash matches => uuid in skipped (not extracted)
