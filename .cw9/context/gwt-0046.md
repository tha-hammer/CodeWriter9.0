# Context for gwt-0046: scanner_python_line_ranges

IMPORTANT: This spec models the line-number assignment invariant of a Python source scanner.
Do NOT model file I/O, directory walking, or param parsing.
The behavior is about LINE NUMBER CORRECTNESS for extracted Skeleton objects.

## Behavior
Given a Python source file with functions at known line positions,
when scan_file is called,
then each Skeleton.line_number equals the 1-indexed line of its def/async def keyword.

## Concrete Data Shapes

```python
@dataclass
class SkeletonParam:
    name: str
    type: str = ""
    is_self: bool = False

@dataclass
class Skeleton:
    function_name: str
    file_path: str
    line_number: int          # 1-indexed, always the def/async def line
    class_name: str | None = None
    visibility: str = "public"
    is_async: bool = False
    params: list[SkeletonParam] = field(default_factory=list)
    return_type: str | None = None
    file_hash: str = ""
```

## Key Constructs Detected
- `def func_name(` — regular function, line_number = line of `def`
- `async def func_name(` — async function, line_number = line of `async`
- Multi-line signatures: scanner gathers continuation lines until parens balance
- The line_number always refers to the FIRST line (where def/async def appears)

## Implementation Pattern
```
i = 0  (0-indexed line counter)
while i < len(lines):
    if line matches _DEF_RE:  # r"^(\s*)(async\s+)?def\s+(\w+)\s*\("
        skeleton.line_number = i + 1  # 1-indexed
        # gather continuation lines j = i+1..N for multi-line params
        # line_number stays at i + 1 regardless of continuation lines
```

## Key Invariants to Model
- LineNumberCorrect: For every skeleton S, S.line_number = 1-indexed position of def/async def
- NoGaps: If a def exists at line L, a skeleton with line_number=L is produced
- MultiLineStable: For multi-line signatures, line_number is always the FIRST line
- OneIndexed: line_number >= 1 always (never 0)

## Modeling Approach
Model as a state machine with:
- CONSTANTS: Lines (sequence of line types: Empty, Comment, Def, AsyncDef, Continuation, Other), MaxLines
- Variables: cursor (0-indexed), skeletons (set of {name, line_num}), in_signature (bool)
- Actions: ScanLine (advance cursor, detect def, record skeleton), GatherContinuation (consume continuation lines without changing line_number)
- Invariants: LineNumberCorrect (every skeleton's line_num = cursor + 1 at time of match)
