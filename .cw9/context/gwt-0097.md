# Context for gwt-0097: imports-gating

## Behavior
Given abstraction_gap has reached a terminal verdict, when `_check_dependencies("imports", results, PASS_DEPENDENCIES)` evaluates whether imports should be scheduled, then imports is scheduled only if abstraction_gap is PASS or WARNING; if abstraction_gap is FAIL or blocked, imports is marked blocked.

## Concrete Data Shapes

```
Verdict == {"pass", "fail", "warning", "blocked"}
PassName == {"artifacts", "coverage", "interaction", "abstraction_gap", "imports"}

PASS_DEPENDENCIES == [
    "imports" |-> {"abstraction_gap"},
]

Results == DICT(PassName -> [verdict: Verdict])

CheckResult == BOOLEAN  \* TRUE = schedule, FALSE = blocked
```

## Key Invariants to Model

- **SingleDependency**: imports scheduled => abstraction_gap.verdict IN {"pass", "warning"}
- **FailBlocks**: abstraction_gap.verdict = "fail" => imports is blocked
- **BlockedCascades**: abstraction_gap.verdict = "blocked" => imports is blocked (transitive from coverage/interaction failures)
- **WarningPermits**: abstraction_gap.verdict = "warning" => imports is scheduled
- **MissingBlocks**: if abstraction_gap not in results, imports is not schedulable

## Modeling Approach

- CONSTANTS: AgVerdict drawn from Verdict
- Variables: results (map of pass name to verdict), imports_scheduled (BOOLEAN)
- Actions: SetAbstractionGapVerdict(v), EvaluateImports
- Invariants: SingleDependency, FailBlocks, BlockedCascades

## Test Interface (MANDATORY)

```python
from registry.cli import _check_dependencies

PASS_DEPENDENCIES = {
    "artifacts": [],
    "coverage": ["artifacts"],
    "interaction": ["artifacts"],
    "abstraction_gap": ["coverage", "interaction"],
    "imports": ["abstraction_gap"],
}

# Test: abstraction_gap pass -> imports scheduled
results = {"abstraction_gap": {"verdict": "pass"}}
assert _check_dependencies("imports", results, PASS_DEPENDENCIES) is True

# Test: abstraction_gap warning -> imports scheduled
results = {"abstraction_gap": {"verdict": "warning"}}
assert _check_dependencies("imports", results, PASS_DEPENDENCIES) is True

# Test: abstraction_gap fail -> imports blocked
results = {"abstraction_gap": {"verdict": "fail"}}
assert _check_dependencies("imports", results, PASS_DEPENDENCIES) is False

# Test: abstraction_gap blocked -> imports blocked
results = {"abstraction_gap": {"verdict": "blocked"}}
assert _check_dependencies("imports", results, PASS_DEPENDENCIES) is False

# Test: abstraction_gap missing -> not schedulable
results = {}
assert _check_dependencies("imports", results, PASS_DEPENDENCIES) is False
```

## Anti-Patterns (DO NOT USE)

- Do NOT model the full dependency chain (coverage -> abstraction_gap -> imports) -- only the direct dependency matters for _check_dependencies
- Do NOT re-verify upstream dependencies -- _check_dependencies only looks at direct parents in PASS_DEPENDENCIES
- Do NOT model concurrent scheduling -- imports has a single linear dependency
- Do NOT treat "unknown" as a valid gating verdict -- the pass must have a terminal verdict
