# Context for gwt-0025: forward_subgraph_complete

IMPORTANT: This spec MUST model a GRAPH TRAVERSAL (breadth-first/DFS reachability).
Do NOT model environment variables, SDK imports, concurrent workers, or extraction counts.
The behavior is about TRANSITIVE CLOSURE over a directed graph of function call edges.

## Behavior
Given records A, B, C where A calls B and B calls C (via ins.source_uuid edges),
when get_forward_subgraph(A.function_name) is called,
then the result contains A, B, and C (full transitive callee closure).

## Data Model
Records are linked via the `ins` table:
- `ins.record_uuid` = the record that HAS this input
- `ins.source_uuid` = the record that PROVIDES this input (the callee)

So if A calls B: there's an ins row with record_uuid=A.uuid, source_uuid=B.uuid.

## SQL Implementation
```sql
WITH RECURSIVE forward(uuid) AS (
    SELECT uuid FROM records WHERE function_name = ?     -- seed
    UNION
    SELECT i.source_uuid                                 -- follow callees
    FROM ins i
    JOIN forward f ON i.record_uuid = f.uuid
    WHERE i.source_uuid IS NOT NULL
)
SELECT * FROM records WHERE uuid IN (SELECT uuid FROM forward)
```

## Key Invariants to Model
- Completeness: every transitively reachable callee is included
- Termination: the CTE terminates even with cycles (SQLite UNION deduplicates)
- Seed correctness: the named function itself is always in the result
- No phantom edges: only follow source_uuid that are NOT NULL

## Modeling Approach
Model as a graph traversal:
- CONSTANTS: Nodes (set of UUIDs), MaxDepth
- Variables: edges (set of {from, to} pairs modeling ins.source_uuid links), visited (set of UUIDs), frontier (set of UUIDs to explore)
- Actions: Seed (add starting node to visited+frontier), Expand (pick from frontier, add its callees), Done (frontier empty)
- Invariants: CompletenessInvariant (when Done, visited contains all reachable nodes), SeedAlwaysIncluded (starting node in visited), TerminationGuaranteed (frontier shrinks or visited grows each step)
