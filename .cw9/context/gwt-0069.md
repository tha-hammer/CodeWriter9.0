# Context for gwt-0069: behavioral_contracts_checked

## Behavior
Given a list of CrossCuttingRule objects and a CrawlStore with dependency edges where some callees match a rule's resource_type but lack required OUT fields,
When check_behavioral_contracts(store, rules) is called,
Then returns a BehavioralReport listing each violated rule-edge pair (with rule name, callee UUID, and missing contract), counting compliant edges as satisfied, and reporting zero violations when no edges match any rule or all matching edges are compliant.

## Concrete Data Shapes

```python
@dataclass
class CrossCuttingRule:
    resource_type: str          # matched against FnRecord metadata or path patterns
    required_outs: list[str]    # OUT field names that must exist
    position: str               # "pre" | "post" | "wrap"

@dataclass
class BehavioralViolation:
    rule_resource_type: str     # which rule was violated
    callee_uuid: str            # the function missing the required OUT
    callee_function: str        # function name for reporting
    missing_contract: str       # the specific required_out that's absent
    caller_uuid: str            # the calling function (for edge context)
    caller_function: str

@dataclass
class BehavioralReport:
    violations: list[BehavioralViolation]
    satisfied: int              # count of edges where all rules were met
    total_checked: int          # total edges that matched any rule
```

Existing types used:
```python
# From crawl_types.py
@dataclass
class FnRecord:
    uuid: str
    function_name: str
    file_path: str
    outs: list[OutField]        # each has .name (OutKind) and .type_str
    # ... other fields

class OutKind(str, Enum):
    OK = "ok"
    ERROR = "error"
    SIDE_EFFECT = "side_effect"
```

## Key Invariants to Model

- ViolationComplete: every violation identifies rule, callee, caller, and missing contract
- NoFalsePositives: if callee has all required outs for matching rules, no violation is reported
- EmptyEdgesEmptyReport: zero dependency edges → zero violations, zero satisfied
- NoMatchNoViolation: rules that match no functions → zero violations, zero satisfied (not counted)
- CountConsistency: total_checked = len(violations) + satisfied

## Modeling Approach

- CONSTANTS: Rules (set of rule records), Edges (set of caller→callee pairs), FnOuts (function → set of out names)
- Variables: violations (set), satisfied_count (Nat), checked_count (Nat), current_edge, phase
- Actions: PickEdge, MatchRule (check if callee's resource_type matches), CheckOuts (verify required outs present), RecordResult
- Invariants: CountConsistency, NoFalsePositives, ViolationComplete

## Test Interface (MANDATORY)

```python
from registry.seam_checker import check_behavioral_contracts, load_cross_cutting_rules
from registry.crawl_types import BehavioralReport, BehavioralViolation, CrossCuttingRule, FnRecord, OutField, OutKind
from registry.crawl_store import CrawlStore

# Setup: create a CrawlStore with test data
store = CrawlStore(":memory:")

# Insert a callee that touches "database" (is_external=True, boundary_contract mentions db)
callee = FnRecord(
    uuid="callee-001",
    function_name="save_record",
    file_path="src/db/store.py",
    class_name=None,
    line_number=10,
    src_hash="abc123",
    is_external=True,
    do_description="Saves record to database",
    outs=[],  # Missing audit_event — will trigger violation
    ins=[],
    failure_modes=[],
    operational_claim="Database write",
    boundary_contract="database write operation",
)
store.upsert_record(callee)

# Insert a caller that depends on the callee
caller = FnRecord(
    uuid="caller-001",
    function_name="process_batch",
    file_path="src/pipeline.py",
    class_name=None,
    line_number=20,
    src_hash="def456",
    is_external=False,
    do_description="Processes a batch",
    outs=[OutField(name=OutKind.OK, type_str="list", description="results")],
    ins=[],  # Will add an IN referencing callee to create edge
    failure_modes=[],
    operational_claim="Batch processor",
)
store.upsert_record(caller)

# Create dependency edge: caller → callee (via ins with source_uuid)
store.conn.execute(
    "INSERT INTO ins (record_uuid, name, type_str, source, source_uuid, ordinal) VALUES (?, ?, ?, ?, ?, ?)",
    ("caller-001", "db_result", "dict", "internal_call", "callee-001", 0),
)
store.conn.commit()

# Test: violation expected (callee has no audit_event out)
rules = [CrossCuttingRule(resource_type="database", required_outs=["audit_event"], position="wrap")]
report = check_behavioral_contracts(store, rules)

assert isinstance(report, BehavioralReport)
assert report.total_checked == report.satisfied + len(report.violations)
assert len(report.violations) >= 1
v = report.violations[0]
assert v.missing_contract == "audit_event"
assert v.rule_resource_type == "database"

# Test: no edges → empty report
empty_store = CrawlStore(":memory:")
report = check_behavioral_contracts(empty_store, rules)
assert report.total_checked == 0
assert report.satisfied == 0
assert len(report.violations) == 0
```

## Anti-Patterns (DO NOT USE)

- Do NOT model CrawlStore internals — abstract edge retrieval as a set of (caller, callee) pairs
- Do NOT use TLA+ sequences for iteration — use set comprehension with \E
- Do NOT model resource_type matching heuristics — abstract to a boolean match function
- Do NOT conflate with type-checking (check_seam) — this is behavioral contract checking, separate concern
