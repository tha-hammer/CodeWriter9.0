# Context for gwt-0037: ProjectContext.from_target() config resolution

## Component
`python/registry/context.py` — `ProjectContext.from_target()` class method (line 68)

## Behavior
`from_target(target_root, engine_root=None)` is a dispatcher with two resolution stages:

1. **Config resolution**: If `engine_root` is None, looks for `<target>/.cw9/config.toml`, reads `[engine].root`, resolves path, checks `candidate.is_dir()`
2. **Auto-detection fallback**: Walks 3 levels up from `context.py` itself, checks for `templates/` subdirectory
3. **Routing**:
   - `engine_root == target_root` → `self_hosting(engine_root)`
   - `engine_root != target_root` → `external(engine_root, target_root)`
   - `engine_root is None` → `installed(target_root)`

## Key Invariant
Config.toml is read only when engine_root is not provided. A missing config file or stale path (not a dir) silently falls through to auto-detection — no error raised.

## State Variables for TLA+ Model
- `config_exists`: boolean — whether .cw9/config.toml exists
- `config_engine_valid`: boolean — whether the configured engine root is a real directory
- `engine_root_provided`: boolean — whether caller passed explicit engine_root
- `engine_equals_target`: boolean — routes to self_hosting vs external
- `auto_detect_success`: boolean — whether parent walk finds templates/
- `selected_mode`: enum {SELF_HOSTING, EXTERNAL, INSTALLED}

## Terminal Condition
Exactly one mode is selected. The mode is deterministic given the input state.
