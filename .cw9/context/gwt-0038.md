# Context for gwt-0038: ProjectContext.external() isolation

## Component
`python/registry/context.py` — `ProjectContext.external()` class method (line 108)

## Behavior
`external(engine_root, target_root)` creates a context where:
- Engine paths (templates, tools, python) come from `engine_root`
- State paths (schema, specs, bridge artifacts, sessions) go under `target_root/.cw9/`
- Test output goes to `target_root/tests/generated/` (outside `.cw9/`)

## Key Invariant
`state_root = target_root / ".cw9"`. Engine resources are read-only from engine_root. State is isolated under target's `.cw9/` directory.

The critical difference from self_hosting:
- `artifact_dir` (`<state_root>/bridge`) != `test_output_dir` (`<target>/tests/generated`)
- `spec_dir` = `<state_root>/specs` (not `templates/pluscal/instances/`)
- `schema_dir` = `<state_root>/schema` (not `<root>/schema/`)

## State Variables for TLA+ Model
- `engine_root`: path (distinct from target_root)
- `target_root`: path
- `state_root`: derived as target_root / ".cw9"
- `artifact_dir_under_state`: boolean — True in external mode, False in self-hosting
- `spec_dir_under_state`: boolean — True in external mode, False in self-hosting

## Terminal Condition
All state paths are under state_root. All engine paths are under engine_root. No cross-contamination.
