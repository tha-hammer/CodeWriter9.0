# Context for gwt-0086: run_review_pass_constructs_query (auto-fix mode)

## Behavior
Given --auto-fix active and a review pass executing, when `_run_review_pass()` constructs the query() call, then json_mode=True is forced and streaming is disabled so that the result text is machine-parseable structured output.

## Concrete Data Shapes

```
ReviewPassConfig == [
    name: STRING,
    prompt_file: PATH,
    plan_path: STRING,
    cwd: STRING,
    json_mode: BOOLEAN,       \* forced TRUE in auto-fix mode
]

ClaudeAgentOptions == [
    allowed_tools: SEQUENCE(STRING),
    system_prompt: STRING,
    max_turns: NAT,
    model: STRING,
]

\* _run_review_pass signature (unchanged):
\* async def _run_review_pass(name, prompt_file, plan_path, cwd, json_mode) -> dict
\*
\* In auto-fix mode, the CALLER sets json_mode=True before calling _run_review_pass.
\* _run_review_pass does not know about auto-fix -- it just respects json_mode.
\*
\* When json_mode=True:
\*   - AssistantMessage text blocks are NOT printed to stdout (streaming suppressed)
\*   - Only ResultMessage is captured
\*   - Result text is expected to be structured (parseable by _extract_findings)

PassResult == [
    name: STRING,
    verdict: {"pass", "fail", "warning", "unknown"},
    session_id: STRING,
    cost_usd: REAL,
    result_length: NAT,
    result_text: STRING,      \* structured JSON when json_mode=True
]
```

## Key Invariants to Model

- **JsonModeForced**: in auto-fix path, json_mode is always TRUE when calling _run_review_pass
- **NoStreaming**: json_mode=TRUE => AssistantMessage text blocks are not written to stdout
- **ResultCaptured**: ResultMessage.result is always captured regardless of json_mode
- **VerdictParsed**: _parse_verdict is called on result_text to determine verdict
- **CostAccumulated**: cost_usd from ResultMessage is always captured

## Modeling Approach

- CONSTANTS: JsonMode (BOOLEAN), AutoFixActive (BOOLEAN)
- Variables: stdout_written (BOOLEAN), result_captured (BOOLEAN), verdict (Verdict)
- Actions: ConstructQuery, ProcessAssistantMessage, ProcessResultMessage
- Invariants: JsonModeForced, NoStreaming, ResultCaptured

## Anti-Patterns (DO NOT USE)

- Do NOT modify _run_review_pass to accept an auto_fix parameter -- json_mode=True is sufficient
- Do NOT model the SDK internals (message types, streaming protocol) -- treat query() as a black box
- Do NOT model the prompt file contents -- treat prompt loading as atomic
- Do NOT conflate "json_mode forces structured output" with "output is valid JSON" -- _run_review_pass does not validate the JSON structure; that is _extract_findings' job
- Do NOT add auto-fix awareness inside _run_review_pass -- the caller controls json_mode
