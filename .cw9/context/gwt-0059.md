# Context for gwt-0059: typescript_profile_file_validity

## Behavior
Given bridge artifacts with verifier conditions, when compile_assertions is called and embedded
in a test file, the file passes tsc --noEmit. Three-stage verification: compile → collect → run.

## MANDATORY PlusCal Template

```tla
---- MODULE typescript_profile_file_validity ----

EXTENDS Integers, FiniteSets, TLC

Verifiers == <<
    [name |-> "inv1", has_cond |-> TRUE,  is_dict |-> TRUE],
    [name |-> "inv2", has_cond |-> TRUE,  is_dict |-> TRUE],
    [name |-> "inv3", has_cond |-> FALSE, is_dict |-> TRUE],
    [name |-> "skip", has_cond |-> TRUE,  is_dict |-> FALSE]
>>
N == 4

(* --algorithm TSFileValidity

variables
    cursor   = 1,
    compiled = {},
    stage    = "idle",
    passed   = FALSE;

define
    BatchConsistent ==
        cursor > N =>
            \A k \in 1..N :
                (Verifiers[k].is_dict /\ Verifiers[k].has_cond) =>
                    \E c \in compiled : c.name = Verifiers[k].name

    NonDictSkipped ==
        ~(\E c \in compiled : c.name = "skip")

    StageOrdering ==
        stage \in {"idle", "compile_all", "type_check", "discover", "run", "done"}
end define;

process verifier = "verifier"
begin
    CompileAll:
        stage := "compile_all";
        while cursor <= N do
            ProcessV:
                if Verifiers[cursor].is_dict /\ Verifiers[cursor].has_cond then
                    compiled := compiled \cup {[name |-> Verifiers[cursor].name]};
                end if;
            AdvanceV:
                cursor := cursor + 1;
        end while;
    TypeCheck:
        stage := "type_check";
        passed := TRUE;
    Discover:
        stage := "discover";
    Run:
        stage := "run";
    Done:
        stage := "done";
end process;

end algorithm; *)

====
```

RULES: Output ONLY the module once. No CONSTANTS. No RECURSIVE. No CHOOSE. No THEOREM after ====.

## Test Interface (MANDATORY)

```python
from registry.lang_typescript import TypeScriptProfile
from registry.lang import CompiledExpression, VerifyResult

profile = TypeScriptProfile()
# verify_test_file returns VerifyResult dataclass:
#   .passed: bool
#   .stage: str ("compile", "collect", or "run")
#   .errors: list[str]
#   .stdout: str
#   .stderr: str

# compile_assertions returns dict[str, CompiledExpression]
# Keys are verifier names from bridge artifacts
```

Anti-Patterns (DO NOT USE — these cause test failures)

- `TypeScriptProfile(some_arg)` ← WRONG. Takes no arguments
- `result.has_error` ← DOES NOT EXIST on CompiledExpression
- `result.get("target_expr")` ← WRONG. It's a dataclass, not a dict
- `result["target_expr"]` ← WRONG. Use attribute access
