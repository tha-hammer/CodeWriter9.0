# Context for gwt-0041: Context stack ranking in test plan prompt

## Component
`python/registry/test_gen_loop.py` — `build_test_plan_prompt()` (line 126)

## Behavior
The test plan prompt assembles context sections in a fixed ranked order:

| Rank | Source | Content |
|------|--------|---------|
| 1 | `ctx.simulation_traces` | TLC `-simulate` output traces |
| 2 | `ctx.api_context` | Import paths + class/method signatures |
| 3 | `ctx.bridge_artifacts["verifiers"]` + `ctx.compiler_hints` | TLA+ conditions + partial expressions |
| 4 | `ctx.tla_spec_text` | Full verified TLA+ module text |
| 5 | `ctx.lang_profile.build_structural_patterns()` | Hard-coded fixture/assertion templates |

Rank 1 is conditional (only if simulation_traces is non-empty). If empty but test_scenarios is non-empty, a fallback "Trace-derived Scenarios" section appears after rank 5.

## Key Invariant
The ranking is fixed. Higher-ranked context always appears before lower-ranked in the prompt. Simulation traces (when present) are always the first context section. The LLM instruction block varies based on whether simulation traces are present.

## State Variables for TLA+ Model
- `has_sim_traces`: boolean
- `has_api_context`: boolean (always true in practice)
- `has_verifiers`: boolean
- `has_tla_spec`: boolean
- `has_test_scenarios`: boolean (fallback for missing sim traces)
- `section_order`: sequence of rank numbers in output order

## Terminal Condition
The section_order is a subsequence of [1,2,3,4,5] respecting the ranking, with each section present iff its source is non-empty.
