# Context for gwt-0033: loop_writes_spec_on_pass

IMPORTANT: This spec models the SPEC FILE PERSISTENCE behavior — on TLC PASS, the verified
.tla and .cfg files are copied to spec_dir, and simulation traces are written. The MODULE name MUST be "LoopWritesSpecOnPass".

## Behavior
`run_loop()` drives the one-shot loop. When TLC verifies successfully (LoopResult.PASS),
three file operations happen atomically:
1. Copy compiled .tla to spec_dir/<gwt-id>.tla
2. Copy .cfg to spec_dir/<gwt-id>.cfg (if exists)
3. Write sim traces to spec_dir/<gwt-id>_sim_traces.json

On FAIL or RETRY-then-FAIL, NO files are written to spec_dir.

## Key Variables
- loop_result: "idle" | "running" | "pass" | "fail"
- compiled_spec_exists: BOOLEAN (temp file exists after compilation)
- cfg_exists: BOOLEAN (companion .cfg file exists)
- spec_dir_has_tla: BOOLEAN (output .tla written)
- spec_dir_has_cfg: BOOLEAN (output .cfg written)
- spec_dir_has_traces: BOOLEAN (sim traces written)
- phase: "start" | "run_loop" | "check_result" | "write_files" | "done"

## Key Invariants
- PassImpliesWrite: phase = "done" /\ loop_result = "pass" => spec_dir_has_tla = TRUE
- FailImpliesNoWrite: phase = "done" /\ loop_result = "fail" => spec_dir_has_tla = FALSE
- CfgConditional: spec_dir_has_cfg = TRUE => cfg_exists = TRUE
- TracesOnPass: phase = "done" /\ loop_result = "pass" => spec_dir_has_traces = TRUE
- BoundedExecution: step_count <= MaxSteps

## PlusCal Constraints — CRITICAL

### One assignment per variable per label
PlusCal forbids assigning the same variable twice within the same label.

### Use finite sets in .cfg
MaxSteps = 10

### No duplicate MODULE headers
Exactly ONE `---- MODULE LoopWritesSpecOnPass ----` and ONE `====`.

### No \* BEGIN TRANSLATION markers
Do NOT include `\* BEGIN TRANSLATION` / `\* END TRANSLATION` markers.

### CRITICAL: Constant classification hint
For any CONSTANT that is used as a set, add `ASSUME X # {}` after the CONSTANTS block.
This tells the TLC cfg generator to treat it as a finite set, not a numeric value.
For this spec, MaxSteps is numeric — no ASSUME needed for it.
