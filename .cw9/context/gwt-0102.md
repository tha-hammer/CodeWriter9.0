# Context for gwt-0102: review-manifest-tracking

## Behavior
Given a review pass completes during a correction loop, when `_run_review_pass()` returns, then `_review_manifest.json` is updated with pass_name, attempt number, verdict, report_path, session_id, and cost. The manifest is created on the first write and appended atomically on subsequent writes.

## Concrete Data Shapes

```
ManifestEntry == [
    pass_name: STRING,
    attempt: NAT,          \* 1-indexed attempt number
    verdict: STRING,       \* "pass", "fail", "warning", "unknown"
    report_path: STRING,   \* absolute path to the review report
    session_id: STRING,    \* SDK session ID for the review
    cost: REAL,            \* cost in USD
]

ReviewManifest == [
    entries: SEQUENCE(ManifestEntry),
]

\* On disk: _review_manifest.json
\* {"entries": [ManifestEntry, ...]}
```

## Key Invariants to Model

- **CreateOnFirst**: manifest file does not exist before first review => created with one entry
- **AppendOnSubsequent**: manifest file exists => new entry appended, existing entries preserved
- **AtomicWrite**: write is atomic (write to temp + rename, or read-modify-write with lock)
- **EntryComplete**: every entry has all 6 fields (pass_name, attempt, verdict, report_path, session_id, cost)
- **AttemptMonotonic**: for a given pass_name, attempt numbers are strictly increasing
- **NoDataLoss**: existing entries are never overwritten or removed when appending

## Modeling Approach

- CONSTANTS: MaxAttempts (1..3), PassNames
- Variables: manifest_exists (BOOLEAN), entries (SEQUENCE of ManifestEntry), file_content (manifest on disk)
- Actions: FirstReviewComplete(entry), SubsequentReviewComplete(entry)
- Invariants: EntryComplete, AttemptMonotonic, NoDataLoss, AtomicWrite

## Test Interface (MANDATORY)

```python
import asyncio, json, os
from unittest.mock import AsyncMock, patch, MagicMock
from pathlib import Path
from registry.cli import _run_review_pass

# Test: first review creates manifest
manifest_path = Path("/tmp/test_manifest/_review_manifest.json")
manifest_path.parent.mkdir(parents=True, exist_ok=True)
if manifest_path.exists():
    manifest_path.unlink()

with patch("registry.cli.query", new_callable=AsyncMock) as mock_q:
    mock_q.return_value = MagicMock(
        result="PASS: all good",
        cost_usd=0.05,
        session_id="sess-001",
    )
    result = asyncio.run(_run_review_pass(
        "artifacts", "prompts/artifacts.md", "/tmp/plan.md", "/tmp/test", True
    ))

assert manifest_path.exists()
data = json.loads(manifest_path.read_text())
assert len(data["entries"]) >= 1
entry = data["entries"][-1]
assert entry["pass_name"] == "artifacts"
assert entry["attempt"] >= 1
assert "verdict" in entry
assert "report_path" in entry
assert "session_id" in entry
assert "cost" in entry

# Test: second review appends without losing first
with patch("registry.cli.query", new_callable=AsyncMock) as mock_q:
    mock_q.return_value = MagicMock(
        result="FAIL: missing coverage",
        cost_usd=0.03,
        session_id="sess-002",
    )
    result = asyncio.run(_run_review_pass(
        "coverage", "prompts/coverage.md", "/tmp/plan.md", "/tmp/test", True
    ))

data = json.loads(manifest_path.read_text())
assert len(data["entries"]) >= 2
```

## Anti-Patterns (DO NOT USE)

- Do NOT overwrite the manifest on each write -- always read-append-write or create fresh
- Do NOT use non-atomic writes (open + write without rename) -- concurrent passes could corrupt the file
- Do NOT store the manifest in memory only -- it must persist to disk for correction agent consumption
- Do NOT omit any of the 6 required fields -- the correction agent depends on all of them
- Do NOT use 0-indexed attempts -- attempt numbers start at 1
