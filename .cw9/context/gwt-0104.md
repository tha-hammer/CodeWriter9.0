# Context for gwt-0104: final-summary-with-cost-breakdown

## Behavior
Given --auto-fix is active and orchestration has completed (all review and correction passes finished), when the final summary is assembled, then it includes a cost object with: review_passes array, correction_passes array, and total sum. Each per-entry object has pass_name, attempt, and cost from query().

## Concrete Data Shapes

```
CostEntry == [
    pass_name: STRING,
    attempt: NAT,         \* 1-indexed
    cost: REAL,           \* USD from query() response
]

CostBreakdown == [
    review_passes: SEQUENCE(CostEntry),
    correction_passes: SEQUENCE(CostEntry),
    total: REAL,          \* sum of all review + correction costs
]

FinalSummary == [
    overall: STRING,      \* "pass", "fail", "warning"
    passed: SET(STRING),
    failed: SET(STRING),
    blocked: SET(STRING),
    cost: CostBreakdown,
]
```

## Key Invariants to Model

- **TotalIsSum**: cost.total = SUM(review_passes[i].cost) + SUM(correction_passes[i].cost)
- **ReviewEntriesComplete**: every review pass that ran has an entry in review_passes with pass_name, attempt, cost
- **CorrectionEntriesComplete**: every correction pass that ran has an entry in correction_passes
- **NoNegativeCost**: every cost value >= 0
- **CostPresent**: final summary always has cost object, even if no corrections ran (correction_passes = [])
- **AttemptTracking**: multiple attempts for the same pass appear as separate entries with incrementing attempt numbers

## Modeling Approach

- CONSTANTS: PassNames, MaxAttempts (1..3)
- Variables: review_costs (SEQUENCE of CostEntry), correction_costs (SEQUENCE of CostEntry), total (REAL), summary (FinalSummary)
- Actions: RecordReviewCost(pass_name, attempt, cost), RecordCorrectionCost(pass_name, attempt, cost), AssembleSummary
- Invariants: TotalIsSum, ReviewEntriesComplete, CorrectionEntriesComplete, NoNegativeCost

## Test Interface (MANDATORY)

```python
import asyncio
from unittest.mock import AsyncMock, patch, MagicMock
from registry.cli import _orchestrate_reviews_with_correction

# Mock query to return controlled costs
call_count = {"n": 0}

async def mock_query(**kwargs):
    call_count["n"] += 1
    return MagicMock(
        result="PASS: ok" if call_count["n"] <= 3 else "FAIL: missing",
        cost_usd=0.05,
        session_id=f"sess-{call_count['n']}",
    )

# Test: summary includes cost breakdown
with patch("registry.cli.query", side_effect=mock_query):
    result = asyncio.run(_orchestrate_reviews_with_correction(
        mode="self",
        plan_path="/tmp/plan.md",
        cwd="/tmp/test",
        phase="all",
        json_mode=True,
        max_retries=2,
    ))
    assert "cost" in result
    cost = result["cost"]
    assert "review_passes" in cost
    assert "correction_passes" in cost
    assert "total" in cost
    assert isinstance(cost["total"], (int, float))
    assert cost["total"] >= 0

    # Each entry has required fields
    for entry in cost["review_passes"]:
        assert "pass_name" in entry
        assert "attempt" in entry
        assert "cost" in entry

    # Total equals sum
    all_costs = [e["cost"] for e in cost["review_passes"]] + [e["cost"] for e in cost["correction_passes"]]
    assert abs(cost["total"] - sum(all_costs)) < 0.001

# Test: no corrections -> correction_passes is empty list
with patch("registry.cli.query", side_effect=mock_query):
    call_count["n"] = 0
    result = asyncio.run(_orchestrate_reviews_with_correction(
        mode="self",
        plan_path="/tmp/plan.md",
        cwd="/tmp/test",
        phase="all",
        json_mode=True,
        max_retries=0,
    ))
    assert result["cost"]["correction_passes"] == [] or len(result["cost"]["correction_passes"]) == 0
```

## Anti-Patterns (DO NOT USE)

- Do NOT omit the cost object when no corrections ran -- always include it with an empty correction_passes array
- Do NOT compute total from manifest alone -- use the actual cost_usd returned by each query() call
- Do NOT use floating point string formatting for cost comparison in tests -- use epsilon-based comparison
- Do NOT flatten review and correction costs into one array -- keep them in separate buckets for clarity
- Do NOT forget to track attempt numbers -- a pass retried 3 times should have 3 separate entries
