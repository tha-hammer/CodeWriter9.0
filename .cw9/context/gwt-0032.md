# Context for gwt-0032: extract_preserves_gwts

IMPORTANT: This spec models DAG MERGE SEMANTICS — extract builds a fresh schema DAG, then merges
with existing behavioral nodes (GWTs, requirements) from the prior DAG file. The MODULE name MUST be "ExtractPreservesGwts".

## Behavior
`cmd_extract` runs SchemaExtractor.extract() which produces a fresh DAG from schema files
(only RESOURCE nodes). The existing DAG file may contain GWT and REQUIREMENT nodes from
prior `cw9 register` calls. The invariant: after extract, all behavioral nodes from the
old DAG must be present in the new DAG alongside the freshly extracted schema nodes.

## Key Variables
- old_dag_nodes: set of node IDs in the existing DAG file (mix of RESOURCE, GWT, REQUIREMENT)
- extracted_nodes: set of node IDs from SchemaExtractor (RESOURCE only)
- result_nodes: set of node IDs in the output DAG
- old_gwt_nodes: subset of old_dag_nodes that are GWT or REQUIREMENT kind
- phase: "load_old" | "extract" | "merge" | "save" | "done"

## Key Invariants
- GwtPreserved: phase = "done" => old_gwt_nodes \subseteq result_nodes
- SchemaFresh: phase = "done" => extracted_nodes \subseteq result_nodes
- NoGwtLoss: phase = "done" => \A n \in old_gwt_nodes : n \in result_nodes
- BoundedExecution: step_count <= MaxSteps

## PlusCal Constraints — CRITICAL

### One assignment per variable per label
PlusCal forbids assigning the same variable twice within the same label.

### Use finite sets in .cfg
Use small finite model value sets, e.g.:
- OldGwts = {g1, g2} (GWT nodes in old DAG)
- OldReqs = {r1} (REQUIREMENT nodes in old DAG)
- SchemaNodes = {s1, s2, s3} (freshly extracted RESOURCE nodes)
- MaxSteps = 10

### No duplicate MODULE headers
Exactly ONE `---- MODULE ExtractPreservesGwts ----` and ONE `====`.

### No \* BEGIN TRANSLATION markers
Do NOT include `\* BEGIN TRANSLATION` / `\* END TRANSLATION` markers.
