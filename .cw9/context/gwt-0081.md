# Context for gwt-0081: orchestrate_reviews_computes_the_overall

## Behavior
Given all applicable review passes have completed with individual verdicts, when `_orchestrate_reviews()` computes the overall status, then if any verdict is 'fail' then status='fail'; else if any verdict is 'warning' then status='warning'; else status='pass'; cmd_plan_review returns exit code 0 for pass/warning and 1 for fail.

## Concrete Data Shapes

```
Verdict == {"pass", "fail", "warning", "unknown"}
Status == {"pass", "fail", "warning"}
ExitCode == {0, 1}

PassResult == [name: STRING, verdict: Verdict, session_id: STRING, cost_usd: REAL, result_length: NAT]

AggregationInput == DICT(STRING -> PassResult)
AggregationResult == [status: Status, exit_code: ExitCode]
```

## Key Invariants to Model

- **FailDominates**: "fail" IN verdicts => status = "fail"
- **WarningSecond**: "fail" NOT IN verdicts AND "warning" IN verdicts => status = "warning"
- **PassRequiresAllPass**: status = "pass" => ALL verdicts are "pass" or "unknown"
- **ExitCodeMapping**: status IN {"pass", "warning"} => exit_code = 0; status = "fail" => exit_code = 1
- **UnknownTreatedAsPass**: "unknown" alone (without fail/warning) does NOT make status fail or warning

## Modeling Approach

- CONSTANTS: NumPasses (1..5)
- Variables: verdicts (sequence of Verdict), status (Status), exit_code (ExitCode)
- Actions: AddVerdict(v), ComputeOverall
- Invariants: FailDominates, WarningSecond, ExitCodeMapping

## Test Interface (MANDATORY)

```python
import asyncio
import argparse
from unittest.mock import patch, AsyncMock
from registry.cli import _aggregate_verdicts, cmd_plan_review, _orchestrate_reviews

# Direct unit test of _aggregate_verdicts
results_all_pass = {
    "artifacts": {"verdict": "pass"},
    "coverage": {"verdict": "pass"},
}
status, exit_code = _aggregate_verdicts(results_all_pass)
assert status == "pass"
assert exit_code == 0

# One fail dominates
results_one_fail = {
    "artifacts": {"verdict": "pass"},
    "coverage": {"verdict": "fail"},
}
status, exit_code = _aggregate_verdicts(results_one_fail)
assert status == "fail"
assert exit_code == 1

# Warning without fail
results_warning = {
    "artifacts": {"verdict": "pass"},
    "coverage": {"verdict": "warning"},
}
status, exit_code = _aggregate_verdicts(results_warning)
assert status == "warning"
assert exit_code == 0

# cmd_plan_review exit code integration
# Build a Namespace matching argparse output
args = argparse.Namespace(
    plan_path="plan.md",
    target_dir="/tmp/test",
    self_hosting=True,
    external=False,
    json_output=True,
    phase="all",
)

async def mock_orchestrate(mode, plan_path, cwd, phase, json_mode):
    return {
        "status": "fail",
        "mode": mode,
        "plan_path": plan_path,
        "phase": phase,
        "timestamp": "2026-01-01T00:00:00+00:00",
        "results": {"artifacts": {"verdict": "fail", "name": "artifacts", "cost_usd": 0.01, "session_id": "s1", "result_length": 10}},

    }

with patch("registry.cli._orchestrate_reviews", side_effect=mock_orchestrate):
    exit_code = cmd_plan_review(args)
    assert exit_code == 1
```

## Anti-Patterns (DO NOT USE)

- Do NOT model "unknown" as a failure -- it is treated like pass for aggregation
- Do NOT model the blocked status here -- that is gwt-0078's concern (artifacts gate)
- Keep the verdict set small -- just model 2-3 passes with different verdicts to cover all cases
- Do NOT test the formatting of the JSON/text output -- only test the status and exit code
