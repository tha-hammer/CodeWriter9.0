# Context for gwt-0076: Rust Entry Point Discovery

## Behavior
Given a Rust project root with skeletons from the scanner and source files on disk,
when the Rust discoverer runs,
then it returns EntryPoint objects for:
- fn main() with no impl block (entry_type=MAIN)
- pub fn in lib.rs with no impl block (entry_type=PUBLIC_API for library codebase type)
- #[get]/#[post]/#[put]/#[delete] attribute macros mapping to the following fn (entry_type=HTTP_ROUTE with route and method)
- Axum .route() builder patterns (entry_type=HTTP_ROUTE)
- #[derive(Parser)] structs (entry_type=CLI_COMMAND)

Dotfile directories are excluded.

## Concrete Data Shapes

```python
@dataclass
class Skeleton:
    function_name: str       # e.g., "main", "new_service", "health_check"
    file_path: str           # e.g., "src/main.rs", "src/lib.rs"
    line_number: int
    class_name: str | None = None  # None = free function; "MyStruct" = impl block
    visibility: str = "public"     # "public" = `pub` keyword; "private" = no pub
    is_async: bool = False
    params: list = field(default_factory=list)
    return_type: str | None = None
    file_hash: str = ""

@dataclass
class EntryPoint:
    file_path: str
    function_name: str
    entry_type: EntryType
    route: str | None = None
    method: str | None = None
```

## Scanner Signals for Rust

| Signal | Meaning |
|---|---|
| `visibility="public"` | `pub` keyword present |
| `visibility="private"` | No `pub` keyword |
| `class_name=None` | Free function (not inside impl block) |
| `class_name="MyStruct"` | Method inside `impl MyStruct` |
| `function_name="main"` | Binary entry point (if class_name is None) |
| `is_async=True` | `async fn` |

## Framework Patterns to Detect

### Actix-web / Rocket Attribute Macros (file scan)
```rust
#[get("/health")]
async fn health() -> impl Responder {
    "ok"
}

#[post("/users")]
async fn create_user(body: web::Json<User>) -> impl Responder { ... }
```
Regex: `#\[(get|post|put|delete|patch|head)\s*\(\s*"([^"]+)"`
Then capture the following fn name: `^\s*(?:pub\s+)?(?:async\s+)?fn\s+(\w+)`

### Axum Router Builder (file scan)
```rust
let app = Router::new()
    .route("/users", get(list_users))
    .route("/users", post(create_user));
```
Regex: `\.route\s*\(\s*"([^"]+)"\s*,\s*(get|post|put|delete|patch|head)\s*\(`

### Clap CLI (file scan)
```rust
#[derive(Parser, Debug)]
#[command(name = "myapp")]
struct Cli {
    #[arg(short)]
    verbose: bool,
}
```
Regex for derive: `#\[derive\([^)]*Parser[^)]*\)\]`
Then capture the following struct name: `\s*(?:pub\s+)?struct\s+(\w+)`

## Key Invariants to Model

- **MainIsTopLevel**: Only `main` functions with `class_name is None` produce MAIN entries
- **MainNotImpl**: `main()` inside an impl block (class_name != None) is NOT a MAIN entry point
- **PublicOnlyForLibrary**: PUBLIC_API entries only produced when codebase_type="library"
- **ImplMethodsExcluded**: Skeletons with class_name != None are never PUBLIC_API entries
- **DotfilesExcluded**: Files under directories starting with "." are excluded
- **AttributeStateReset**: After an attribute macro (#[get(...)]), if the next line is not a fn definition, the pending state resets (no stale matching)
- **AxumMethodUppercase**: Axum route methods are stored uppercase ("GET", "POST")
- **ActixMethodUppercase**: Actix attribute methods are stored uppercase ("GET", "POST")
- **DeduplicatedResults**: No two entry points share the same (file_path, function_name) pair
- **ClapDetectsStruct**: Clap Parser detection identifies the struct name, not the derive attribute

## Test Interface (MANDATORY)

```python
from registry.entry_points import discover_entry_points
from registry.crawl_types import EntryPoint, EntryType, Skeleton
from pathlib import Path

def _skel(file_path, function_name, *, visibility="public", class_name=None):
    return Skeleton(
        function_name=function_name,
        file_path=file_path,
        line_number=1,
        visibility=visibility,
        class_name=class_name,
    )

# main() detection
skels = [_skel("src/main.rs", "main")]
eps = discover_entry_points(tmp_path, "cli", lang="rust", skeletons=skels)
assert len(eps) == 1
assert eps[0].entry_type == EntryType.MAIN

# impl main excluded
skels = [_skel("src/lib.rs", "main", class_name="App")]
eps = discover_entry_points(tmp_path, "cli", lang="rust", skeletons=skels)
assert len(eps) == 0

# pub fn as public API
skels = [
    _skel("src/lib.rs", "new_service", visibility="public"),
    _skel("src/lib.rs", "internal_helper", visibility="private"),
]
eps = discover_entry_points(tmp_path, "library", lang="rust", skeletons=skels)
names = [ep.function_name for ep in eps]
assert "new_service" in names
assert "internal_helper" not in names

# Actix-web attribute route detection
src = tmp_path / "src"
src.mkdir()
(src / "main.rs").write_text('#[get("/health")]\nasync fn health() -> impl Responder {\n    "ok"\n}\n')
eps = discover_entry_points(tmp_path, "web_app", lang="rust", skeletons=[])
assert any(ep.route == "/health" for ep in eps)

# Axum router detection
(src / "main.rs").write_text('.route("/users", get(list_users))\n.route("/users", post(create_user))\n')
eps = discover_entry_points(tmp_path, "web_app", lang="rust", skeletons=[])
routes = [ep.route for ep in eps]
assert "/users" in routes
```

## Anti-Patterns (DO NOT USE)

- Do NOT match `fn main` inside strings or comments (acceptable false positive rate for regex)
- Do NOT require lib.rs specifically for PUBLIC_API — any .rs file with pub fn qualifies
- Do NOT carry pending_route state across files — reset per file
- Do NOT match `#[derive(Parser)]` without checking the next struct line
- Do NOT scan hidden directories (paths with components starting with ".")

## TLA+ Modeling Hints

- Do NOT introduce new CONSTANTS beyond what the harness provides — TLC has no values for them and throws Java exceptions
- Use variables initialized with `\in` for nondeterministic choices: `codebase_type \in {"web_app", "cli", "library"}`, boolean flags `\in BOOLEAN`
- Model skeletons as a count (integer), not as individual records — iterate with index
- Use `either/or` for nondeterministic skeleton properties (public vs private, has class_name vs None, function named "main" vs not)
- Keep the model small: 2-3 skeletons, 2-3 source files
