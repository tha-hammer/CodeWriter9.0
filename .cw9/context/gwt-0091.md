# Context for gwt-0091: After correction, review re-runs

## Behavior

Given a correction agent has completed for a failed pass, when _run_correction_pass() returns, then the same review pass is re-run, the attempt counter increments, and the new verdict/cost/report are tracked independently from prior attempts.

## Concrete Data Shapes

```python
# History after correction + re-review
pass_state = {
    "name": "coverage",
    "attempt": 2,                # incremented after correction
    "max_retries": 2,
    "verdict": "PASS",           # new verdict from attempt 2
    "history": [
        # Attempt 1: FAIL
        {"name": "coverage", "verdict": "FAIL", "session_id": "s1", "cost_usd": 0.05, "result_length": 200, "result_text": "FAIL: ..."},
        # Attempt 2: PASS (after correction)
        {"name": "coverage", "verdict": "PASS", "session_id": "s2", "cost_usd": 0.04, "result_length": 180, "result_text": "All good"},
    ],
    "corrections": [
        {"name": "coverage_correction", "session_id": "sc1", "cost_usd": 0.07, "result_length": 300, "result_text": "Fixed..."},
    ],
}

# Total cost = sum of all history + corrections costs
# total_cost = 0.05 + 0.04 + 0.07 = 0.16
```

## Key Invariants to Model

- **AttemptIncrements**: After correction returns, the attempt counter increments by exactly 1 before re-review.
- **SamePassReRun**: The re-run invokes the same review pass (same name, same prompt_file, same plan_path). It is not a different pass.
- **IndependentTracking**: Each attempt produces a distinct result dict with its own session_id, cost_usd, verdict, and result_text. Results are appended to history, not overwritten.
- **CostAccumulation**: Total cost for a pass is the sum across all attempts and corrections. No result is discarded.
- **VerdictFromLatest**: The pass's final verdict is taken from the most recent attempt, not aggregated across attempts.
- **HistoryLength**: len(history) == attempt after the loop completes. len(corrections) == number of FAIL attempts that had attempt < max_retries.

## Modeling Approach

Model the attempt sequence as a list that grows monotonically. Each loop iteration appends one review result. If correction occurs, it also appends to a corrections list. The verdict variable is overwritten each iteration (only the latest matters for scheduling). The attempt counter is a simple incrementing natural number.

Key state variables:
- `attempt`: Nat (starts at 1, increments after each correction+re-review cycle)
- `history`: Seq(ReviewResult) -- append-only
- `corrections`: Seq(CorrectionResult) -- append-only
- `currentVerdict`: verdict from history[Len(history)]
- `totalCost`: Real (sum of all costs)

Invariant: `Len(history) == attempt` after each review completes.
Invariant: `Len(corrections) <= Len(history) - 1` (at most one correction per failed attempt, and the last attempt has no correction yet).

## Anti-Patterns (DO NOT USE)

- Do NOT overwrite the previous attempt's result. Each attempt is appended to history as a separate entry.
- Do NOT reuse the session_id from a prior attempt. Each review and correction call produces a unique session_id.
- Do NOT take the verdict from any attempt other than the latest. Earlier verdicts are historical only.
- Do NOT increment the attempt counter before correction. The sequence is: review (attempt N) -> correction -> increment -> review (attempt N+1).
- Do NOT aggregate costs only from the final attempt. All attempts and corrections contribute to total cost.
