# Context for gwt-0066: seam_unresolved_flagged

IMPORTANT: This spec MUST model the detection of UNRESOLVED dependency edges.
Do NOT model type checking (that's gwt-0064/0065) or the full report (gwt-0067).
The behavior is about FLAGGING inputs with source=internal_call but source_uuid=NULL.

## Behavior
Given crawl.db has callee B with ins [{name: "data", source: "internal_call", source_uuid: NULL}],
when check_all_seams() iterates the deps view and unresolved ins,
then an UnresolvedSeam is reported for B.ins["data"] because source_uuid is NULL
and the seam cannot be verified.

## Concrete Data Shapes

```python
from dataclasses import dataclass
from registry.crawl_types import DispatchKind

@dataclass
class SeamMismatch:
    caller_uuid: str        # empty string for unresolved
    callee_uuid: str
    caller_function: str    # empty string for unresolved
    callee_function: str
    callee_input_name: str
    expected_type: str
    provided_type: str      # empty string for unresolved
    dispatch: DispatchKind
    severity: str            # "unresolved" for this GWT

@dataclass
class SeamReport:
    mismatches: list[SeamMismatch]     # severity="type_mismatch" items
    unresolved: list[SeamMismatch]     # severity="unresolved" items
    satisfied: int
    total_edges: int
```

The CrawlStore has these relevant queries:
- `deps` view: `SELECT dependent_uuid, dependency_uuid, dispatch, dispatch_candidates FROM ins WHERE source_uuid IS NOT NULL` — these are RESOLVED edges
- Unresolved: `SELECT * FROM ins WHERE source = 'internal_call' AND source_uuid IS NULL` — these CANNOT be checked

## Key Invariants to Model
- UnresolvedDetected: every ins row with source=internal_call and source_uuid=NULL produces exactly one SeamMismatch with severity="unresolved"
- NoFalseUnresolved: ins rows with source_uuid IS NOT NULL are never flagged as unresolved
- UnresolvedInReport: all unresolved items end up in SeamReport.unresolved, not in SeamReport.mismatches

## Modeling Approach
- CONSTANTS: Inputs (set of input records), MaxInputs
- Variables: inputs_checked (set), unresolved_found (set), phase (scanning/done)
- Actions: ScanInput (check if source_uuid is null), FlagUnresolved, SkipResolved
- Invariants: AllScanned (when done, inputs_checked = Inputs), UnresolvedCorrect (unresolved_found = {i in Inputs : i.source_uuid = NULL AND i.source = "internal_call"})

## Test Interface (MANDATORY)

```python
from registry.seam_checker import check_all_seams
from registry.crawl_store import CrawlStore
import tempfile, pathlib

# Create a temporary crawl.db with one unresolved internal_call
with tempfile.TemporaryDirectory() as td:
    db_path = pathlib.Path(td) / "crawl.db"
    store = CrawlStore(db_path)
    store.connect()

    # Insert a callee record with an unresolved internal_call input
    from registry.crawl_types import FnRecord, InField, OutField, InSource, OutKind
    callee = FnRecord(
        uuid="bbbb-2222", function_name="process_data", file_path="b.py",
        src_hash="def",
        ins=[InField(
            name="data", type_str="dict", source=InSource.INTERNAL_CALL,
            source_uuid=None, source_function="unknown_fn"
        )],
        do_description="processes data",
        outs=[OutField(name=OutKind.OK, type_str="bool")]
    )
    store.upsert_record(callee)

    report = check_all_seams(store)
    assert len(report.unresolved) == 1
    assert report.unresolved[0].severity == "unresolved"
    assert report.unresolved[0].callee_input_name == "data"
    store.close()
```

## Anti-Patterns (DO NOT USE)
- Do NOT model type compatibility checks — unresolved edges skip type checking entirely
- Do NOT query deps view for unresolved items — deps only has RESOLVED edges (source_uuid IS NOT NULL)
- Do NOT use check_seam for unresolved edges — check_seam requires both caller and callee records
- Do NOT confuse source_uuid=NULL with source_function=NULL — source_function may exist even when source_uuid is NULL (backfill failed to resolve)
