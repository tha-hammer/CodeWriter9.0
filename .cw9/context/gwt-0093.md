# Context for gwt-0093: Failed pass blocks dependents

## Behavior

Given a pass is permanently FAIL or already blocked, when _check_dependencies() is evaluated for dependent passes, then dependent passes are skipped (never run), marked "blocked", and all transitive downstream passes are also blocked.

## Concrete Data Shapes

```python
PASS_DEPENDENCIES = {
    "artifacts": [],
    "coverage": ["artifacts"],
    "interaction": ["artifacts"],
    "abstraction_gap": ["coverage", "interaction"],
    "imports": ["abstraction_gap"],
}

# Results dict tracking pass outcomes
results = {
    "artifacts": {"verdict": "PASS"},
    "coverage": {"verdict": "FAIL"},       # permanent fail
    "interaction": {"verdict": "PASS"},
}

# _check_dependencies("abstraction_gap", results, PASS_DEPENDENCIES)
# coverage is FAIL -> returns False -> abstraction_gap blocked

# After blocking propagation
final_results = {
    "artifacts": {"verdict": "PASS"},
    "coverage": {"verdict": "FAIL"},
    "interaction": {"verdict": "PASS"},
    "abstraction_gap": {"verdict": "blocked"},  # blocked by coverage
    "imports": {"verdict": "blocked"},           # transitively blocked by abstraction_gap
}
```

## Key Invariants to Model

- **DirectBlock**: If any direct dependency has verdict "FAIL" or "blocked", the dependent pass is blocked.
- **TransitiveBlock**: Blocking propagates transitively. If A blocks B and B blocks C, then C is blocked even though A is not a direct dependency of C.
- **NeverRun**: A blocked pass is never executed. _run_review_pass() is never called for it.
- **BlockedIsDistinct**: "blocked" is a distinct status from "FAIL". FAIL means the pass ran and could not be fixed. Blocked means the pass was never attempted.
- **PartialSuccess**: Passes with no failed ancestors proceed normally even when siblings are blocked.
- **CheckBeforeRun**: _check_dependencies() is called before scheduling any pass. If it returns False, the pass is skipped.

## Modeling Approach

Model the dependency graph as a DAG with PASS_DEPENDENCIES defining edges. For each pass, before scheduling, check all direct dependencies. A dependency is satisfied if its verdict is in {"PASS", "WARNING"}. A dependency is unsatisfied if its verdict is in {"FAIL", "blocked"}. If any dependency is unsatisfied, mark the pass "blocked" and propagate.

Use a topological ordering to process passes. This ensures that by the time we check a pass's dependencies, all upstream passes have been resolved (completed, failed, or blocked).

Key state variables:
- `passStatus`: function from pass_name -> {"pending", "PASS", "WARNING", "FAIL", "blocked"}
- `scheduled`: set of pass names that have been dispatched
- `blocked`: set of pass names marked blocked

Invariant: `\A p \in blocked: \E dep \in dependencies[p]: passStatus[dep] \in {"FAIL", "blocked"}`
Invariant: `\A p \in scheduled: \A dep \in dependencies[p]: passStatus[dep] \in {"PASS", "WARNING"}`

## Test Interface (MANDATORY)

```python
from registry.cli import _check_dependencies

PASS_DEPENDENCIES = {
    "artifacts": [],
    "coverage": ["artifacts"],
    "interaction": ["artifacts"],
    "abstraction_gap": ["coverage", "interaction"],
    "imports": ["abstraction_gap"],
}

# Direct block: coverage FAIL blocks abstraction_gap
results_direct = {
    "artifacts": {"verdict": "PASS"},
    "coverage": {"verdict": "FAIL"},
    "interaction": {"verdict": "PASS"},
}
assert _check_dependencies("abstraction_gap", results_direct, PASS_DEPENDENCIES) == False

# Transitive block: abstraction_gap blocked -> imports blocked
results_transitive = {
    "artifacts": {"verdict": "PASS"},
    "coverage": {"verdict": "FAIL"},
    "interaction": {"verdict": "PASS"},
    "abstraction_gap": {"verdict": "blocked"},
}
assert _check_dependencies("imports", results_transitive, PASS_DEPENDENCIES) == False

# No block: all deps satisfied
results_ok = {
    "artifacts": {"verdict": "PASS"},
    "coverage": {"verdict": "PASS"},
    "interaction": {"verdict": "WARNING"},
}
assert _check_dependencies("abstraction_gap", results_ok, PASS_DEPENDENCIES) == True

# No deps: artifacts always eligible
assert _check_dependencies("artifacts", {}, PASS_DEPENDENCIES) == True
```

## Anti-Patterns (DO NOT USE)

- Do NOT run a blocked pass and then discard the result. Blocked passes must never be invoked.
- Do NOT treat "blocked" as "FAIL" for summary purposes. They are distinct: FAIL ran and failed, blocked never ran.
- Do NOT check only direct dependencies. Transitive blocking must be handled (either eagerly via propagation or by checking the resolved status of each direct dep).
- Do NOT assume all passes after a failure are blocked. Only passes with the failed pass in their transitive dependency set are blocked. Sibling passes with independent deps proceed.
