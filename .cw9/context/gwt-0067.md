# Context for gwt-0067: seam_report_complete

IMPORTANT: This spec MUST model the COMPLETENESS INVARIANT of the seam report.
Do NOT model individual type checking or unresolved detection in detail.
The behavior is about ACCOUNTING FOR EVERY EDGE: mismatches + unresolved + satisfied = total.

## Behavior
Given crawl.db with N dependency edges, M type mismatches, K unresolved,
and (N-M-K) satisfied seams,
when check_all_seams() runs to completion,
then SeamReport contains exactly M mismatches plus K unresolved plus (N-M-K) satisfied
and total_edges equals N so every edge is accounted for.

## Concrete Data Shapes

```python
from dataclasses import dataclass
from registry.crawl_types import DispatchKind

@dataclass
class SeamMismatch:
    caller_uuid: str
    callee_uuid: str
    caller_function: str
    callee_function: str
    callee_input_name: str
    expected_type: str
    provided_type: str
    dispatch: DispatchKind
    severity: str            # "type_mismatch" | "unresolved" | "no_ok_output"

@dataclass
class SeamReport:
    mismatches: list[SeamMismatch]     # severity="type_mismatch" or "no_ok_output"
    unresolved: list[SeamMismatch]     # severity="unresolved"
    satisfied: int                      # count of compatible seams
    total_edges: int                    # N = len(mismatches) + len(unresolved) + satisfied
```

The total_edges count includes:
1. Resolved edges from `deps` view (source_uuid IS NOT NULL) — each becomes either a mismatch or satisfied
2. Unresolved edges from `ins` where source='internal_call' AND source_uuid IS NULL

## Key Invariants to Model
- Completeness: total_edges = len(mismatches) + len(unresolved) + satisfied
- Partition: every edge is classified into exactly one of {mismatch, unresolved, satisfied} — no edge is counted twice, no edge is missed
- NonNegative: satisfied >= 0, total_edges >= 0
- MonotonicProgress: as edges are processed, the sum mismatches + unresolved + satisfied monotonically increases toward total_edges

## Modeling Approach
- CONSTANTS: Edges (set of dependency edges), MaxEdges
- Variables: processed (set of edges already classified), mismatches (set), unresolved (set), satisfied (nat), total (nat), phase (init/processing/done)
- Actions: InitReport (set total = |Edges|), ProcessResolvedEdge (classify as mismatch or satisfied), ProcessUnresolvedEdge (add to unresolved), FinalizeReport
- Invariants: CompletenessHolds (when phase=done: |mismatches| + |unresolved| + satisfied = total), PartitionCorrect (mismatches ∩ unresolved = {} AND processed ⊆ Edges), MonotonicProgress (|mismatches| + |unresolved| + satisfied <= total at all times)

## Test Interface (MANDATORY)

```python
from registry.seam_checker import check_all_seams
from registry.crawl_store import CrawlStore
from registry.crawl_types import FnRecord, InField, OutField, InSource, OutKind
import tempfile, pathlib

with tempfile.TemporaryDirectory() as td:
    db_path = pathlib.Path(td) / "crawl.db"
    store = CrawlStore(db_path)
    store.connect()

    # Insert caller A with ok output "str"
    caller = FnRecord(
        uuid="aaaa-1111", function_name="get_name", file_path="a.py",
        src_hash="abc", ins=[], do_description="returns name",
        outs=[OutField(name=OutKind.OK, type_str="str")]
    )
    store.upsert_record(caller)

    # Insert callee B1 — satisfied seam (str matches str)
    callee1 = FnRecord(
        uuid="bbbb-2222", function_name="use_name", file_path="b.py",
        src_hash="def",
        ins=[InField(name="name", type_str="str", source=InSource.INTERNAL_CALL, source_uuid="aaaa-1111")],
        do_description="uses name",
        outs=[OutField(name=OutKind.OK, type_str="bool")]
    )
    store.upsert_record(callee1)

    # Insert callee B2 — mismatch (str vs int)
    callee2 = FnRecord(
        uuid="cccc-3333", function_name="count_items", file_path="c.py",
        src_hash="ghi",
        ins=[InField(name="count", type_str="int", source=InSource.INTERNAL_CALL, source_uuid="aaaa-1111")],
        do_description="counts items",
        outs=[OutField(name=OutKind.OK, type_str="int")]
    )
    store.upsert_record(callee2)

    # Insert callee B3 — unresolved (source_uuid=NULL)
    callee3 = FnRecord(
        uuid="dddd-4444", function_name="process_data", file_path="d.py",
        src_hash="jkl",
        ins=[InField(name="data", type_str="dict", source=InSource.INTERNAL_CALL, source_uuid=None, source_function="unknown")],
        do_description="processes data",
        outs=[OutField(name=OutKind.OK, type_str="bool")]
    )
    store.upsert_record(callee3)

    report = check_all_seams(store)

    # Completeness invariant: total = mismatches + unresolved + satisfied
    assert report.total_edges == len(report.mismatches) + len(report.unresolved) + report.satisfied
    assert report.total_edges == 3  # 1 satisfied + 1 mismatch + 1 unresolved
    assert report.satisfied == 1
    assert len(report.mismatches) == 1
    assert len(report.unresolved) == 1
    store.close()
```

## Anti-Patterns (DO NOT USE)
- Do NOT model individual type checking logic — abstract it as "classify edge as mismatch or satisfied"
- Do NOT confuse resolved edge count with total edge count — total includes unresolved
- Do NOT assume deps view returns all edges — it only returns RESOLVED edges
- Do NOT make satisfied a list — it's a count (int), not a list of SeamMismatch objects
