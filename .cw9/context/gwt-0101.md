# Context for gwt-0101: correction-agent-scope-constraints

## Behavior
Given a correction agent is executing against cw9_plan_fix.md for any pass type, when the agent decides what modifications to make, then it may only modify context files, plan text, and test scaffolding; it must NOT modify source code, TLA+ specs, or bridge artifacts. The scope of corrections varies by pass type: coverage adds assertions to test scaffolding; abstraction_gap copies the decision checklist into context; imports removes dead imports and flags wrong abstractions; artifacts lists pipeline steps to re-run.

## Concrete Data Shapes

```
PassType == {"artifacts", "coverage", "interaction", "abstraction_gap", "imports"}

ModifiableTarget == {"context_files", "plan_text", "test_scaffolding"}
ForbiddenTarget == {"source_code", "tla_specs", "bridge_artifacts"}

CorrectionAction == [
    pass_type: PassType,
    target: ModifiableTarget,
    description: STRING,
]

PassCorrectionMap == [
    "coverage" |-> [target: "test_scaffolding", action: "add_assertions"],
    "abstraction_gap" |-> [target: "context_files", action: "copy_decision_checklist"],
    "imports" |-> [target: "context_files", action: "remove_dead_imports_flag_wrong_abstractions"],
    "artifacts" |-> [target: "plan_text", action: "list_pipeline_steps_to_rerun"],
]
```

## Key Invariants to Model

- **NoSourceModification**: correction agent NEVER modifies files in python/registry/ (source code)
- **NoSpecModification**: correction agent NEVER modifies files in .cw9/specs/ or templates/pluscal/
- **NoBridgeModification**: correction agent NEVER modifies *_bridge_artifacts.json files
- **CoverageScope**: pass_type = "coverage" => modifications target test scaffolding (add assertions)
- **AbstractionGapScope**: pass_type = "abstraction_gap" => modifications target context files (copy decision checklist)
- **ImportsScope**: pass_type = "imports" => modifications target context files (remove dead imports, flag wrong abstractions)
- **ArtifactsScope**: pass_type = "artifacts" => modifications target plan text (list pipeline steps to re-run)

## Modeling Approach

- CONSTANTS: PassTypes, ModifiableTargets, ForbiddenTargets
- Variables: pass_type (PassType), modifications (SET of [target, action])
- Actions: CoverageCorrection, AbstractionGapCorrection, ImportsCorrection, ArtifactsCorrection
- Invariants: NoSourceModification, NoSpecModification, NoBridgeModification, per-pass scope constraints

## Test Interface (MANDATORY)

```python
import asyncio
from unittest.mock import AsyncMock, patch, MagicMock
from registry.cli import _run_correction_pass

# The scope constraints are enforced by the system_prompt and allowed_tools.
# Test that the system_prompt includes scope restrictions per pass type.

captured_args = {}

async def mock_query(**kwargs):
    captured_args.update(kwargs)
    return MagicMock(result="corrected", cost_usd=0.02)

# Test: coverage correction prompt mentions test scaffolding / assertions
with patch("registry.cli.query", side_effect=mock_query):
    asyncio.run(_run_correction_pass(
        pass_name="coverage",
        findings={"missing_assertions": ["test_foo"]},
        plan_path="/tmp/plan.md",
        cwd="/tmp/test",
        correction_prompt_file="prompts/cw9_plan_fix.md",
    ))
    prompt = captured_args.get("prompt", "")
    assert "assertion" in prompt.lower() or "test" in prompt.lower()

# Test: artifacts correction prompt mentions pipeline steps
with patch("registry.cli.query", side_effect=mock_query):
    asyncio.run(_run_correction_pass(
        pass_name="artifacts",
        findings={"missing_steps": ["bridge", "gen-tests"]},
        plan_path="/tmp/plan.md",
        cwd="/tmp/test",
        correction_prompt_file="prompts/cw9_plan_fix.md",
    ))
    prompt = captured_args.get("prompt", "")
    assert "pipeline" in prompt.lower() or "re-run" in prompt.lower() or "step" in prompt.lower()
```

## Anti-Patterns (DO NOT USE)

- Do NOT let the correction agent edit .py files under python/registry/ -- it fixes the plan, not the code
- Do NOT let the correction agent regenerate TLA+ specs -- that requires `cw9 loop`
- Do NOT let the correction agent modify bridge artifacts -- those are mechanically generated
- Do NOT conflate "test scaffolding" with "generated tests" -- scaffolding is the structure, not the verified test content
- Do NOT give all pass types the same correction instructions -- each pass has specific correction actions
