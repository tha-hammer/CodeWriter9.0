# Context for gwt-0082: cmd_plan_review_dispatches_to_the_orches

## Behavior
Given cw9 plan-review invoked without --auto-fix, when cmd_plan_review() dispatches to orchestration, then `_orchestrate_reviews()` is called (not `_orchestrate_reviews_with_correction()`); backward compatible with existing behavior.

## Concrete Data Shapes

```
CLIArgs == [
    plan_path: STRING,
    target_dir: STRING,
    self_hosting: BOOLEAN,
    external: BOOLEAN,
    json_output: BOOLEAN,
    phase: {"pre", "post", "all"},
    auto_fix: BOOLEAN,       \* NEW: defaults to FALSE
    max_retries: NAT | NULL, \* NEW: only relevant when auto_fix=TRUE
]

DispatchTarget == {"orchestrate_reviews", "orchestrate_reviews_with_correction"}
```

## Key Invariants to Model

- **DefaultDispatch**: auto_fix = FALSE => dispatch target = "orchestrate_reviews"
- **BackwardCompatible**: when auto_fix is absent from args (old CLI), behavior is identical to current
- **NoCorrection**: auto_fix = FALSE => _orchestrate_reviews_with_correction is never called
- **SameSignature**: _orchestrate_reviews receives (mode, plan_path, cwd, phase, json_mode) -- unchanged

## Modeling Approach

- CONSTANTS: AutoFix (BOOLEAN)
- Variables: dispatch_target (DispatchTarget), called_function (STRING)
- Actions: ParseArgs, Dispatch
- Invariants: DefaultDispatch, BackwardCompatible, NoCorrection

## Test Interface (MANDATORY)

```python
import argparse
from unittest.mock import patch, AsyncMock, MagicMock
from registry.cli import cmd_plan_review, _orchestrate_reviews

# Args without --auto-fix (backward compatible)

    plan_path="plan.md",
    target_dir="/tmp/test",
    self_hosting=True,
    external=False,
    json_output=True,
    phase="all",
    auto_fix=False,
    max_retries=2,
)

mock_result = {

    "mode": "self",
    "plan_path": "plan.md",
    "phase": "all",
    "timestamp": "2026-01-01T00:00:00+00:00",

    "total_cost_usd": 0.01,

# Verify _orchestrate_reviews is called, not _orchestrate_reviews_with_correction
```

## Anti-Patterns (DO NOT USE)

- Do NOT model the orchestration internals -- just model which function gets dispatched to
- Do NOT model the CLI argument parsing itself -- that is gwt-0083/0085's concern
- Do NOT test the output formatting -- only test that the correct function is called
- Do NOT conflate this with the auto_fix=True path -- keep the two dispatch paths separate
