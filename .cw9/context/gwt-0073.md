# Context for gwt-0073: Entry Point Discovery Dispatch

## Behavior
Given `discover_entry_points()` is called with a non-Python `lang` and optional `skeletons` list,
when the language is go, rust, javascript, or typescript,
then the call is dispatched to the corresponding language-specific discoverer.
When lang is python or None, existing Python behavior is preserved unchanged.
When lang is an unknown value, an empty list is returned safely.

## Current Code (entry_points.py:148-177)

```python
def discover_entry_points(
    root: Path, codebase_type: str | None = None, *, lang: str | None = None,
) -> list[EntryPoint]:
    if lang and lang != "python":
        return []  # <-- THE GATE: all non-Python returns empty
    if codebase_type is None:
        codebase_type = detect_codebase_type(root)
    entry_points: list[EntryPoint] = []
    if codebase_type == "web_app":
        entry_points.extend(_discover_web_routes(root))
    elif codebase_type == "cli":
        entry_points.extend(_discover_cli_commands(root))
    elif codebase_type == "event_driven":
        entry_points.extend(_discover_event_handlers(root))
    entry_points.extend(_discover_main_functions(root))
    if codebase_type == "library" or not entry_points:
        entry_points.extend(_discover_public_api(root))
    return entry_points
```

## Key Invariants to Model

- **PythonPreserved**: When lang is None or "python", the output must be identical to the current implementation (same dispatch to _discover_web_routes, _discover_cli_commands, etc.)
- **NonPythonDispatched**: When lang is one of {"go", "rust", "javascript", "typescript"}, the call reaches the corresponding discoverer function (not the Python path)
- **UnknownSafe**: When lang is a string not in the known set (e.g., "ruby", "java"), an empty list is returned without error
- **SkeletonsOptional**: The skeletons parameter defaults to None; callers that don't pass it see no behavioral change
- **CodebaseTypeAutoDetect**: If codebase_type is None, detect_codebase_type(root, lang=lang) is called before dispatch

## Anti-Patterns (DO NOT USE)

- Do NOT change the existing Python helper functions (_discover_web_routes, etc.)
- Do NOT make skeletons a required parameter — existing callers must work unchanged
- Do NOT move codebase_type auto-detection after language dispatch — it must happen first for all languages
- Do NOT import language-specific discoverers from separate modules — they live in entry_points.py
